# OZtone

OZtone（オズトーン）は、C++とWin32 APIで構築された、極限までストイックなデスクトップアクセサリ風メディアプレイヤーです。

「画面上には音楽とビジュアルと最低限のUIだけが存在し、必要な時だけUIが影から現れる」というコンセプトに基づいて設計されています。DirectCompositionによる完全なデスクトップ透過合成と、独自実装の高解像度FFTビジュアライザを備え、作業の邪魔をせずにデスクトップを彩ります。

## ✧ コンセプト & 特徴

* **真のボーダーレス＆透過ウィンドウ**: `DirectComposition` を用いたアルファブレンドにより、Windowsのデスクトップ環境に完全に溶け込む半透明・枠なしウィンドウを実現。
* **ステルスUI**: 通常時はアルバムアートとビジュアライザ、曲情報、シークバーのみを表示。再生コントロールやプレイリストなどは、マウスホバー時のみ滑らかなイージングアニメーションと共にフェードイン / スライドインします。
* **高解像度リアルタイム・ビジュアライザ**: `miniaudio` のオーディオキャプチャと自作FFT（サイズ: 8192）を連携。
  * **Prism Beat**: 音楽のグルーヴに追従する、7色ネオンの心電図風オシロスコープ。
  * **Halo Dust**: 曲名のハッシュから固有のテーマカラーを生成し、ビートに合わせて3Dパーティクルが飛び散る円形スペアナ。
* **スマートなプレイリスト管理**:
  * 複数プレイリスト(`.ozl`)の階層化管理とシステムトレイからの動的切り替え。
  * `TagLib` とバックグラウンドスレッドによる非同期のメタデータ解析・TSVキャッシュ機能。次曲のシームレスなWIC画像プリフェッチ対応。
  * ファイルのドラッグ＆ドロップ対応（SHIFTキー押下でBGMを止めずにキューへ追加可能）。
* **ストイックなカスタマイズ性**: 設定画面は存在しません。すべての設定（レイアウト座標、色、フォント、影の濃さなど）は `OZtone.ini` で管理され、**保存した瞬間に再起動なしでUIへ反映されるホットリロード機構**を搭載しています。
* **グローバル操作対応**: `WH_KEYBOARD_LL` を用いた低レベルキーボードフックにより、アプリがバックグラウンド（非アクティブ）であってもWindowsメディアキーの入力を完璧にフックします。

## ✧ UI・画面構成

OZtoneのUIは、設定（`OZtone.ini`）によって完全に非表示にしたり、オフセット位置をピクセル単位で調整することが可能です。

* **背景**: アルバムアートをウィンドウサイズにフィット＆トリミング。視認性確保のためのダークオーバーレイとビジュアライザが合成されます。
* **左上 (アプリアイコン)**: ドロップ領域兼メニューアンカー。ホバーすると「ロゴ拡張メニュー」が展開し、アプリ終了、プレイリスト固定、ビジュアライザ切替、シャッフル切替、プレイリスト配置切替などの機能へアクセスできます。
* **左下 (Track Info)**: 現在再生中のアルバムアートのサムネイル、曲名、アーティスト名、トラック番号を表示します。
* **下部 (コントロール)**: ホバー時のみ浮かび上がるシークバーと、再生コントロール・音量コントロール（マウスホイール対応）を備えます。
* **右端 / 左端 (プレイリスト)**: 画面端にホバーするとスライドインする階層化プレイリスト。曲一覧とプレイリストファイル一覧を行き来でき、ダブルクリックで対象の曲へジャンプ再生します。

## ✧ サポートフォーマット

* **Audio**: `.mp3`, `.flac`, `.wav`, `.ogg` (Powered by miniaudio)
* **Experimental**: `.mp4`, `.m4a` (OSのMedia Foundationフォールバックによる再生)

## ✧ 技術スタック / アーキテクチャ

* **言語**: C++20
* **フレームワーク / API**: Win32 API
* **グラフィックス**: Direct2D, DirectWrite, WIC (Windows Imaging Component), DirectComposition, DXGI
* **オーディオ**: miniaudio (シングルヘッダライブラリ)
* **メタデータ解析**: TagLib
* **ビルドツール**: CMake
* **アーキテクチャ設計**:
  * ロジックと描画の完全な分離（`LayoutCalculator` によるピクセル計算の独立化）。
  * UIコンポーネントの `Widget` 化による責務分離とFacadeパターン。
  * IPC (`WM_COPYDATA`) を用いた多重起動時のファイルパス転送機能。
