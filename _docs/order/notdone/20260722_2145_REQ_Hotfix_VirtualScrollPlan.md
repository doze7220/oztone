###### 【実装要件】
本タスクでは、左下のアルバムアート・曲名・アーティスト名（TrackInfo）領域において、マウスホイールでトラックをパラパラと遷移させる「仮想スクロールモード」を実装する。

*   **要件1: ホバー判定とアフォーダンス (Window / Widget層)**
    *   TrackInfo領域のホバー判定を追加する（判定領域のX座標は画面左端〜プレイリスト判定領域の手前までとし、プレイリストよりも優先順位を低く設定する）。
    *   ホバー時に既存の非対称カラーフェードと、維持ディレイ（離脱ディレイ）を適用する。
    *   ホバー時、`VolumeControlWidget` で使用しているものと同じデザインの「吹き出しツールチップ」を、タイトル・アーティストの左上付近に表示する。描画ロジックは必ず共通化し、表示オフセット位置はINI設定から取得できるようにする。
*   **要件2: アキュムレータと初動の重み付け (Window層)**
    *   マウスホイール（`WM_MOUSEWHEEL`）入力を受け付け、ホイール上＝前の曲、下＝次の曲とする。
    *   入力を蓄積するアキュムレータを実装する。初動は一定の蓄積値に達するまで反応せず、一度スクロール状態に入ったら蓄積なしで制限なく連続で回せる「高級オーディオのダイヤルのような手触り」を実現する。
*   **要件3: 仮想スクロールとドラムの連動 (Application層)**
    *   ホイール回転中は、現在の音楽を停止せずにそのまま流し続け、`TrackDrum`のアニメーションだけを回す。
    *   ファイルロックを回避するため、仮想スクロール中（ドラム回転中）は画像やタグの先読み・ファイルロード発注を**一切行わない**。
*   **要件4: スクロール確定と自動曲送りのブロック (Application層)**
    *   仮想スクロール状態の最中は、曲が終了しても勝手に次の曲へ進まないように、自動曲送りロジックをブロックする。
    *   ホイールの回転が完全に止まり、一拍（マジックナンバーで約500ms程度）待機した瞬間に、目標曲に対して正式に再生確定（`PlayCurrentTrack`等を発動し、新曲再生と背景切り替えを開始）させる。
