#### 【実装要件】
ロゴ拡張メニュー（LogoMenuWidget）がスライド展開している最中にマウスホバーが外れた際、アニメーションが途中で不自然に停止してしまう不具合を修正する。

*   **要件1: アニメーション進行条件の修正**
    *   `src/Widget_LogoMenu.cpp` の `UpdateAnimation` メソッドを確認する。
    *   現在、スライド進行度（`m_menuProgress` 等）を 1.0f に向かって加算（展開）する条件が `ctx.isLogoMenuHovered` のみに依存している場合、ホバーが外れて維持ディレイ（`m_leaveTimer > 0.0f`）に入った瞬間に更新計算がスキップされ、メニューが途中で止まってしまう。
    *   この加算（展開）の条件を `if (ctx.isLogoMenuHovered || m_leaveTimer > 0.0f)` などのように修正し、維持ディレイ中であればマウスが外れていても「アニメーションを最後まで展開し切る」ようにロジックを改善すること。
    *   `m_leaveTimer <= 0.0f` になった際（ディレイ終了時）に、0.0f に向かって減算（格納アニメーション）を開始する処理は既存のまま維持すること。
