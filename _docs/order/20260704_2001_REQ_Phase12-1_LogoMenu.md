#### 【実装要件】
本タスクの目的は、左上のアプリアイコンにホバーした際、右側へ5つの機能アイコンが easeOut で一斉にスライド展開する拡張メニューを実装することである。アイコンの並び順は配列で管理し、アニメーションと同期したタイピング演出を組み込む。

* **要件1: UIデザイン、配列管理、および INI設定 (LayoutCalculator / ConfigManager / Widget)**
  * アプリアイコンの右側へ展開する5つのボタンを配置する（Y座標はロゴの下端合わせ）。
  * トレイメニューの `TRAY_MENU_ORDER` のように、配列（アイコン文字やコマンドIDをまとめた構造体の配列）でメニューの並び順を管理し、後からコード上で簡単に順番を入れ替えられるアーキテクチャとすること。
  * 現在の想定順: アプリ終了(❌), 再生中の曲削除(🗑️), プレイリスト固定(📜), ビジュアライザ切替(📽️), シャッフルON/OFFダミー(🔀)
  * メニュー付近にテキスト描画用のレイアウト領域を確保する。
  * `ConfigManager` に `[Layout_LogoMenu]` セクションを新設し、MenuIconSize (サイズ), MenuIconSpacing (間隔), MenuScrollDuration (展開完了時間), フォント、色などをすべてINIで管理できるようにする。
* **要件2: easeOut アニメーションとタイピング演出 (UpdateAnimation / Widget)**
  * ホバー状態に応じた展開アニメーションを `UpdateAnimation` で計算する。`MenuScrollDuration` を用いて進行度（0.0f〜1.0f）を算出し、`easeOut` イージングを適用すること。
  * **スライドの仕様**: 全てのアイコンがOZロゴの右端（同じタイミング・同じ発射位置）から飛び出し、同時にそれぞれの目標座標へ到着する動きとする。
  * 展開進行度に同期し、以下の文字列をタイピング描画する（文字数 * progress）。
    * 文字列: `Beyond the r'ai'nbow awaits the game you envisioned in your lullaby days.`
  * トグル機能（📜, 🔀など）が「OFF」の時は、アイコンを半透明（グレーアウト）にし、上に「赤い斜線」を描画するクールな表現を組み込む。
  * 絵文字アイコン自体はカラー設定を無効化し、白単色のシルエットで描画すること。
* **要件3: インタラクション (Window / Application)**
  * ホバー判定領域を展開メニュー部分に動的拡張し、操作中に閉じないようにする。
  * メニュー展開中は、メニュー領域全域を含めてプレイリスト左側配置の時に優先処理とする。
  * 各ボタンのクリックを検知し、対応する処理を実行する（プレイリスト固定機能は、ON時にリストが常時展開されるようにWindow側のホバーロジック等を上書き制御する。※シャッフル機能の裏側はダミーでUI切替のみ）。

#### 【絶対遵守ルール (Constraints)】
* **アーキテクチャの厳守**: レイアウト計算は必ず `LayoutCalculator` で行い、状態更新は `UpdateAnimation` に集約し、描画は `Widget` 内で行うという Phase 9 で確立した責務分離を絶対に守ること。
