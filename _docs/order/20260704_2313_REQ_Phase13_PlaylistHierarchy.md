#### 【実装要件】
本タスクの目的は、プレイリストUI（`PlaylistWidget`）を「曲一覧モード」と「プレイリスト一覧モード」の2階層に拡張し、最上部に固定のツールバーを設けてシームレスなリスト管理を実現することである。

* **要件1: 階層化モードとツールバー領域の導入 (PlaylistWidget / LayoutCalculator)**
  * `PlaylistWidget` に現在のモードを示すフラグ（例: `bool m_isListViewMode` = false）を追加し、曲一覧とプレイリスト一覧の状態を管理する。
  * プレイリスト描画領域の最上部をスクロールしない「固定ツールバー領域」として確保し、スクロール領域はその下から開始するようにレイアウト計算を修正する。
* **要件2: モード別ツールバーの描画と機能 (PlaylistWidget / Window / Application)**
  * ツールバーには3つのボタン（アイコン等）を配置する。
  * **【曲一覧モード時】**
    * 1. `上の階層へ` (📂等): クリックで `m_isListViewMode` を true にし、プレイリスト一覧モードへ遷移する。
    * 2. `曲削除` (🗑️等): 再生中の曲をプレイリストから削除する。
    * 3. `全曲削除` (🧹等): プレイリスト内の全曲をクリアする（既存の `ClearPlaylist()` 相当）。
  * **【プレイリスト一覧モード時】**
    * 1. `上の階層へ`: これ以上上の階層がないため、グレーアウト・斜線等でクリック無効であることを明示する。
    * 2. `新規作成` (➕等): 既存の `CreateNewPlaylist()` を呼び出し、新規プレイリストを作成する。
    * 3. `リスト削除` (🗑️等): クリックした（または現在選択されている）プレイリストファイルを物理削除し、リストを更新する。
* **要件3: ホバー説明テキストの動的表示 (PlaylistWidget)**
  * アフォーダンス向上のため、ツールバーのボタンにマウスがホバーされた際、ツールバー領域の空きスペースに以下の機能説明テキストを **日本語** で描画すること。
  * **【曲一覧モード時】**
    * 上の階層へ (📂) -> `プレイリスト一覧を開く`
    * 曲削除 (🗑️) -> `再生中の曲をプレイリストから削除する`
    * 全曲削除 (🧹) -> `プレイリストの全曲を削除する`
  * **【プレイリスト一覧モード時】**
    * 上の階層へ (📂グレーアウト) -> (テキスト非表示とする)
    * 新規作成 (➕) -> `プレイリストを新規作成する`
    * リスト削除 (🗑️) -> `プレイリストを削除する`
* **要件4: プレイリスト一覧モードの描画と操作 (PlaylistWidget / Application)**
  * `m_isListViewMode` が true の場合、`ConfigManager::GetAvailablePlaylists()` で取得したプレイリストファイル一覧を描画する。
  * ファイル名をタイトルとして描画し、選択中（現在ロードされているプレイリスト）のものはハイライト表示を行う。
  * 項目をクリックした場合、既存の `SwitchPlaylist()` を呼び出してプレイリストを切り替え、自動的に「曲一覧モード」へ戻る（`m_isListViewMode = false`）こと。

#### 【絶対遵守ルール (Constraints)】
* **責務の分離**: ツールバーやリストアイテムのクリック判定は `Window` のコールバック拡張等を活用し、既存のアーキテクチャ（入力検知と描画の分離）を崩さないこと。
* 既存の裏側ロジック（`SwitchPlaylist`, `CreateNewPlaylist`, `GetAvailablePlaylists`, `ClearPlaylist`）を最大限流用し、処理の重複を避けること。
