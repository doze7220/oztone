【目的】
Phase 8-8 最後の微調整: シークバーの横幅指定を「ウィンドウ幅の90%などの割合」から「左右の固定ピクセルマージン」に変更し、UIの余白を安定させます。

【作業手順（厳守事項）】
1. 以下の【実装要件】を満たすタスクリストを作成し、作業ログ（_docs/logs/20260702_0215_Phase8-8_SeekBarMargin.md）を新規作成して保存してください。
2. タスクリストの作成が終わったら、絶対にコードの修正は行わずに、まずはチャットで「タスクリストの作成が完了しました。タスク1を実行してよいか指示をお願いします」と報告し、私の承認を待つこと。

【実装要件】
1. ConfigManager の修正
対象: src/ConfigManager.h, src/ConfigManager.cpp
- [Layout_SeekBar] セクションに以下のパラメータを追加し、読み書きとゲッターを実装してください。
  - SeekBarMargin (左右の固定マージン。デフォルト: 20.0f)
- もし既存の幅割合指定パラメータ（SeekBarWidthRatio 等）があれば、今後使用しないため、可能であれば削除（または無効化）して設定をクリーンアップしてください。

2. Renderer の描画処理の修正
対象: src/Renderer.cpp
- Render メソッド内のシークバー領域（背景バーおよび進行度バー）の描画において、X座標と幅の計算を以下のように修正してください。
  - 開始X座標: `SeekBarMargin` の値
  - シークバーの幅: `logicalWidth - (SeekBarMargin * 2.0f)`
- 進行度（progress）に応じた現在位置バーの幅計算も、上記の新しい幅を基準に行うように修正してください。
- 右端の時間テキスト（timeString）の描画位置や、シークバーに関するホバー領域等の計算でこの幅に依存している箇所があれば、同様にマージンベースの計算に修正してください。
