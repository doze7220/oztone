#### 【実装要件】
本タスクの目的は、システムトレイメニューを拡張し、現在日時を用いた新しいプレイリストファイルを自動生成して「空の状態」で切り替える機能（新規プレイリスト作成）を実装することである。

* **要件1: システムトレイメニューの拡張 (Window)**
  * `WM_TRAYICON` のメニュー構築処理 (`TRAY_MENU_ORDER` 配列等) を改修し、「新規プレイリスト作成 (New Playlist)」メニュー項目を追加する。
* **要件2: ファイル名の自動生成と重複回避 (Application / ConfigManager)**
  * メニューから「新規作成」が実行された際、現在のシステム日時を取得し `playlist_YYYYMMDD_HHMM.lst` (※拡張子は現在の実装に準拠) というファイル名を生成する。
  * 同名ファイルが既にディレクトリに存在する場合は、末尾に `_1`, `_2` などのシーケンスを付与して重複を回避する安全なロジックを組み込むこと。
  * 保存ディレクトリは、現在 `ConfigManager` が管理しているデフォルトのプレイリストパスと同階層とする。
* **要件3: プレイリストの切り替えと初期化 (Application / PlaylistManager)**
  * 生成した新しいファイルパスを `ConfigManager` の `DefaultPlaylistPath` に上書き設定し、INIファイルへ保存する。
  * 既存の `ClearPlaylist()` 相当の処理を実行し、現在の再生を停止、メモリ上のキューと先読み状態をクリアする。
  * 空の状態のまま `PlaylistManager::SaveToFile` を呼び出し、新規プレイリストファイルの実体をディスク上に作成する。
  * ウィンドウのUIを "No Track" の初期状態へリセットする。

#### 【絶対遵守ルール (Constraints)】
* **スコープの厳守**: 今回は「新規プレイリスト作成」のみをターゲットとする。サブメニュー化（ネスト化）や、既存ファイルの自動認識機能などにはまだ手を出さないこと。
