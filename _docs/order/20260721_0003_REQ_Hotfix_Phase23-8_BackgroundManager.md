#### 【実装要件】
`src/BackgroundManager.cpp` (および該当ヘッダ) における画像セットおよびアニメーション更新ロジックを、以下の正しい「スワップ式ステートマシン」へと完全に改修する。

*   **要件1: 新規画像セット時の「フライングスワップ」と「初期化」の完全禁止**
    *   外部から新規画像がセットされるメソッド（例: `SetNewImage` など）の内部で、**「OLDへのスワップ処理」や「NEWのフレーミング情報（X, Y, Scale）の初期値リセット処理」を絶対に行ってはならない**。該当する誤ったコードを全削除すること。
    *   新規画像がセットされた際は、単に `NEW` 用のビットマップ変数に画像を保持し、クロスフェード進行度（`m_fadeProgress`）を `0.0f` にリセットしてフェードを開始するだけに留めること。
    *   ※フレーミング情報（X, Y, Scale）は外部（Application層）から個別のセッター経由で正しい値がセットされるため、BackgroundManager 側で勝手に `0` や `1.0f` に上書きして握り潰してはならない。

*   **要件2: 1.0f 到達時の「スワップ（格上げ）」処理の正しい実装**
    *   `UpdateAnimation` メソッドにおいて、`m_fadeProgress` が `1.0f` に到達した**その瞬間**に、以下のスワップ処理を実行すること。
        1. `NEW` が保持している画像ポインタおよびフレーミング情報（Scale, OffsetX, OffsetY）を、すべて `OLD` の変数へコピー（上書き格上げ）する。
        2. コピー完了後、`NEW` の画像ポインタを `nullptr` にして解放・空状態とする。
        3. （※これにより、フェード完了後は常に OLD レイヤーのみが 100% の不透明度で描画され続ける正しいレイヤー構造となる）
    *   旧実装に存在した「1.0f 到達時に OLD をただ解放・リセットする」という誤ったロジックは全削除すること。

#### 【絶対遵守ルール (Constraints)】
*   **フレーミングデータの絶対保護**: Applicationから渡された背景のフレーミングオフセットやスケール値を、BackgroundManagerがライフサイクルの途中で勝手にゼロリセットしてはならない。
