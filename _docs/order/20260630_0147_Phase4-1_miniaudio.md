**【今回の目的：Phase 4】**
音声処理ライブラリ `miniaudio` を導入し、バックグラウンドでのMP3再生エンジン（AudioPlayerクラス）を構築します。今回はテストとして、起動時に特定のMP3ファイルを再生させます。

**【実装要件】**
1. **AudioPlayer クラスの作成**:
   - `src/AudioPlayer.h` および `src/AudioPlayer.cpp` を新規作成してください。
   - `AudioPlayer.cpp` の先頭で `#define MINIAUDIO_IMPLEMENTATION` を定義してから `#include "miniaudio.h"` をインクルードし、エンジンとサウンド（`ma_engine` 等）をカプセル化して管理してください。
   - 以下のメソッドを実装してください。
     - `bool Initialize()`: miniaudioエンジンの初期化。
     - `void Uninitialize()`: エンジンの解放。
     - `bool Play(const std::string& filepath)`: 指定されたMP3ファイルの再生（今回は単純な再生のみでOK）。
2. **Application への統合**:
   - `Application` クラスのメンバに `AudioPlayer` を追加してください。
   - `Application::Initialize` 内で `AudioPlayer::Initialize` を呼び出し、成功したら `AudioPlayer::Play("assets/test.mp3")` を呼び出してテスト用のMP3を再生させてください。
   - アプリ終了時に適切に `Uninitialize` を呼び出してリソースを解放してください。
3. **CMakeLists.txt の修正**:
   - `AudioPlayer.cpp` をビルド対象に追加してください。
   - 実行時に `assets/test.mp3` が読み込めるよう、ビルド後のカスタムコマンドで `assets/test.mp3` を出力ディレクトリへコピーする処理を追加してください。

**【事後処理】**
実装完了後、`.\build.bat` を実行して自動ビルドテストを行い、エラーがあれば自己修正してください。成功したら `PROJECT_ARCHITECTURE.md` の更新と、`_docs/logs/YYYYMMDD_HHMM_Phase4.md` へ実装レポートを出力してください。
