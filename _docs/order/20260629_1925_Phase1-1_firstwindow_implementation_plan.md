# OzCurtain 実装計画

C++とWin32 APIを使用して、ご要望の仕様に基づいたデスクトップアクセサリ風MP3プレイヤー「OzCurtain」のベースアーキテクチャおよび機能実装計画を提案します。

## 概要と基本方針
- **言語・環境**: C++20, Windows API (Win32)
- **ビルドシステム**: CMake (Visual Studio等での開発を想定)
- **描画エンジン**: Direct2D / DirectWrite (高速な2D描画、半透明処理、綺麗なテキスト描画のため)
- **音声・タグ処理**: 
  - 案A (推奨): `miniaudio` (音声再生とFFT解析用ヘッダオンリーライブラリ) + `TagLib` (MP3のID3タグ・アルバムアート取得用)
  - 案B: Windows Media Foundation (OS標準機能のみで実装、外部依存なしだが実装難易度高)
- **ウィンドウ管理**: フレームレスウィンドウ (`WS_POPUP`)。ドラッグ移動は特定のUI領域（背景など）を掴んだときのメッセージ処理で実装。
- **タスクトレイ常駐**: `Shell_NotifyIcon` を使用した通知領域アイコンの配置とコンテキストメニュー。

> [!NOTE]
> 現在はプロジェクトにドキュメントファイルしか存在しないため、ゼロからのプロジェクト構築となります。

## 提案するプロジェクト構造

```text
ozcurtain/
 ├── CMakeLists.txt         # CMakeビルド設定
 ├── src/
 │   ├── main.cpp           # エントリポイント(WinMain)
 │   ├── Application.cpp    # アプリケーションクラス（メインループ管理）
 │   ├── Window.cpp         # ウィンドウ生成・メッセージプロシージャ
 │   ├── AudioPlayer.cpp    # MP3再生・ビジュアライザデータ取得
 │   ├── Renderer.cpp       # Direct2Dによる描画管理
 │   ├── UIManager.cpp      # 再生ボタン、シークバー、リスト等のUI管理
 │   ├── Config.cpp         # 設定(INI等)の読み書き
 │   └── DropTarget.cpp     # ファイルドラッグ＆ドロップ受け入れ処理
 ├── include/               # ヘッダファイル群
 ├── res/                   # リソースファイル（アイコン等）
 └── _docs/                 # 既存のドキュメント
```

### 各コンポーネントの設計方針

#### 1. UI・描画関連 (Renderer, Window)
- ウィンドウ生成時に枠なしフラグを指定。WM_NCHITTESTをフックし、特定の領域（ドラッグ可能領域）以外は `HTCLIENT` を返し、ドラッグ対象領域は `HTCAPTION` を返すことで擬似的なウィンドウ移動を実現します。
- 描画はDirect2Dを使用し、毎フレーム（あるいはタイマー駆動で）背景アート、ビジュアライザ、UI要素をZオーダー順に重ねて描画します。

#### 2. 音声再生・プレイリスト関連 (AudioPlayer)
- ドラッグ＆ドロップされたファイルのパスを受け取り、プレイリストに追加します。
- シャッフル・リピート機能は内部のインデックス管理で実装します。
- 再生中のファイルからPCMデータを読み取り、FFT（高速フーリエ変換）を行ってビジュアライザ用の周波数データをRendererに渡します。

#### 3. コンフィグ・タスクトレイ (Config, Taskbar)
- タスクトレイのアイコン右クリックから「コンフィグウィンドウ」を開く、あるいは「終了」を選ぶメニューを表示します。
- コンフィグウィンドウは別ダイアログとして作成し、設定変更時にメインウィンドウ（表示位置や描画スタイルなど）へ即座に反映させます。

## 検証計画 (Verification Plan)

### 手動検証
1. プロジェクトをビルドし、起動確認。タスクトレイにアイコンが表示され、設定画面が開けるか。
2. ドラッグ＆ドロップでMP3ファイルを追加し、再生・次/前の曲へスキップできるか。
3. アルバムアートの表示、およびビジュアライザが再生に合わせて動くか。
4. UI操作（メディアキー含む）が正常に反応するか。
5. 設定画面からのサイズ・位置変更が即座に反映されるか。
