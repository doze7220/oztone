#### 【実装要件】
肥大化したPhase 21-8の実装を整理するため、プレイリストのデータ構造からトラックドラムの描画データを完全に切り離す。また、Phase 21-8の過程で追加された場当たり的なハックコードを完全にパージ（削除）し、堅牢なバケツリレーアーキテクチャを再構築する。
※なお、シャッフル状態変更時の対応とドキュメント更新はPhase 21-10で行うため、本フェーズのスコープ外とする。

*   **要件1: ドラム専用データ構造と操作意図の定義**
    *   `Renderer.h` 等に、描画に必要な純粋なデータのみを持つ構造体 `DrumSlotData` (曲名、アーティスト、画像等) を定義する。
    *   ドラムの回転意図を指示する列挙型 `DrumMoveType` (`Next`, `Prev`, `Jump`, `CrossPlaylist`, `Reset` 等) を定義し、`SetTrackInfo` の引数として受け取れるようインターフェースを改修する。
    *   `Application` 層の各操作（メディアキー、ジャンプ、UIクリア等）から、適切な `DrumMoveType` を渡すように連携させる。
*   **要件2: 旧ハックコードの完全パージ（詳細な削除対象の明記）**
    *   計画書内には、Phase 21-8 で追加された以下のハックロジックを **「削除対象」として具体的に記載** すること。
        *   `m_drumPosition` を `-1.0` 等へ強制ワープさせていた処理
        *   `m_oldTrackIndex` に `-1` 等の仮想インデックスを代入・偽装していた処理
        *   文字列 (`"NO TRACK"`) や `-1`、`isResetUI`、`m_oldIsCrossPlaylist` 等のフラグによるバケツリレー空打ちのガード処理
        *   描画時の `size_t` と `int` のキャスト比較によるOLD判定ハック
*   **要件3: 独立したバケツリレーと位置リセットの再構築**
    *   `Renderer` 内に `m_oldDrumSlot` と `m_nowDrumSlot` を実体として保持する。
    *   `SetTrackInfo` 呼び出し時、渡された `DrumMoveType` の意図に従ってのみ現在位置（`m_drumPosition`）を正確にワープさせ、`m_nowDrumSlot` を `m_oldDrumSlot` へコピーしてデータを退避させる絶対的なバケツリレーを構築する。
*   **要件4: Widget_TrackInfo の描画純化**
    *   仮想スロット描画ループにおいて、OLDスロットとNOWスロットを描画する際は絶対にプレイリストデータ（`ctx.shuffleMetadataList` 等）を参照せず、`ctx` を経由して渡された `oldDrumSlot` と `nowDrumSlot` の実体のみを使用してテキストと画像を描画するように純化する。
