#### 【実装要件】
以下の4つの不具合を修正すること。

*   **1. MP4除外対応 (src/Application.cpp)**
    *   `Application::OnFilesDropped` 内にある、拡張子が `.mp4`, `.m4a` の場合にマジックナンバー検証をスキップする実験的コード（`// [EXPERIMENTAL] MP4/M4A Support` 付近）をコメントアウトし、プレイリストへ追加されないようにする。
*   **2. ビジュアライザの音量依存解消 (src/AudioPlayer.cpp 等)**
    *   現在、`ma_engine_config.onProcess` でキャプチャしている音声データがマスターボリューム適用後のものであるため、音量を下げるとビジュアライザも小さくなってしまう。
    *   FFTに渡す前の波形データに対して、現在の音量（`GetVolume()`）を利用して元の振幅レベルに補正（逆算）するか、ボリューム適用前のノードから波形を取得するように処理を改修し、アプリの音量設定に依存せず元データのダイナミクスでビジュアライザが駆動するようにする。（※音量0の際のゼロ除算に注意すること）
*   **3. ヘイローダスト（円形ビジュアライザ）の画面サイズ連動 (src/Visualizer.cpp)**
    *   `Visualizer::DrawCircleParticle` におけるレーザーやパーティクルの速度・飛距離のパラメータが定数（`constexpr`）で固定されているため、リサイズ時にスケールしない。
    *   描画領域（`drawRect.width` や `drawRect.height`）と基準サイズ（例えば 1024.0f）との比率からスケール係数を算出し、放出されるレーザーの長さ、速度、パーティクルの移動量などに乗算することで、ウィンドウサイズに連動した演出に修正する。
*   **4. 設定初期化時の自動再起動 (src/Window.cpp)**
    *   システムトレイからの「設定を初期化（`ID_TRAY_RESET_ALL`）」を実行した際、現状は `WM_CLOSE` が発行されてそのまま終了してしまう。
    *   `WM_CLOSE`（または同等の終了処理）を実行する直前に、`GetModuleFileNameW` で自身のファイルパスを取得し、`ShellExecuteW` 等を用いて新しいプロセスとしてアプリを起動（再起動）する処理を追加する。
