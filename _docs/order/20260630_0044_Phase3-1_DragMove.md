**【今回の目的：Phase 3 - Step 1】**
ウィンドウの完全枠なし化（アクセサリ化）と、左上のアプリアイコン領域を利用した「ドラッグ移動」および「仮の右クリック終了機能」を実装します。

**【実装要件】**
1. **完全枠なし化（ConfigManagerとWindowの調整）**:
   - `ConfigManager` において、ウィンドウのタイトルバーや境界線を表示する設定（例：`ShowTitleBar`, `ShowBorder`等）のデフォルト値を `0` (非表示) に変更し、初回起動時から完全な枠なし（`WS_POPUP`）になるようにしてください。
2. **操作パネル化（Windowクラスの拡張）**:
   - `WindowProc` にて `WM_LBUTTONDOWN` および `WM_RBUTTONDOWN` メッセージを捕捉してください。
   - クリックされたクライアント座標を取得し、`ConfigManager` から取得した `[Layout_AppLogo]` のアイコン矩形領域内（X, Y, Width, Height）にあるかを判定してください（判定処理はインラインで書かず、専用のメソッドとして分離すること）。
   - **ドラッグ移動**: 左クリック（`WM_LBUTTONDOWN`）がアイコン領域内だった場合、`ReleaseCapture()` を呼び出した後、`SendMessage(hwnd, WM_NCLBUTTONDOWN, HTCAPTION, 0)` を送信し、OS標準のタイトルバードラッグ移動をエミュレートしてください。
   - **右クリック終了**: 右クリック（`WM_RBUTTONDOWN`）がアイコン領域内だった場合、`PostMessage(hwnd, WM_CLOSE, 0, 0)` を送信して安全にアプリケーションを終了させてください。
3. **プロジェクト憲法の厳守**:
   - WndProcが肥大化しないよう、状態判定やメッセージ送信のロジックは美しくカプセル化してください。

**【事後処理】**
実装完了後、`.\build.bat` を実行して自動ビルドテストを行い、エラーがあれば自己修正してください。成功したら `PROJECT_ARCHITECTURE.md` の更新と、`_docs/logs/YYYYMMDD_HHMM_Phase3_Step1.md` へ実装レポートを出力してください。
