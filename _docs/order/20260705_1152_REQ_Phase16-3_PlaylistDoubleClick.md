#### 【実装要件】
現在、プレイリスト内の項目（曲やプレイリストファイル）をシングルクリックするだけで即座に再生や切り替えが実行される仕様を改め、「シングルクリック＝項目の選択（フォーカス）」「ダブルクリック＝決定（再生・切り替えの実行）」という挙動に変更する。

* **要件1: ダブルクリックイベントの捕捉 (Window)**
  * `Window::Initialize` におけるウィンドウクラス（`WNDCLASSEXW`）の登録時、`style` に `CS_DBLCLKS` を追加し、ダブルクリックメッセージを受け取れるようにする。
  * `Window::WindowProc` にて `WM_LBUTTONDBLCLK` を捕捉し、プレイリスト領域内でのダブルクリック時に発火するコールバック `SetPlaylistDoubleClickCallback` を新設・実装する。
* **要件2: 選択（フォーカス）状態の管理と描画 (Application / Widget)**
  * シングルクリック（`WM_LBUTTONDOWN`）時は再生を実行せず、「現在フォーカスされているインデックス（`m_focusedPlaylistIndex` など）」を状態として保持するよう `Application` 側を改修する。
  * 保持したフォーカスインデックスを `WidgetContext` を経由して `PlaylistWidget` に渡し、描画時にフォーカスされた項目の背景を薄くハイライト表示（既存の再生中ハイライトとは異なる色・透明度など）して視覚的なフィードバックを与える。
  * ※曲一覧モードとプレイリスト一覧モードの両方で、このフォーカス管理を機能させること。
* **要件3: 決定ロジックのダブルクリックへの移行 (Application)**
  * 既存のシングルクリックコールバックで行っていた `JumpToIndex` や `SwitchPlaylist` の呼び出しロジックを、新設したダブルクリックコールバック側へと移行する。
