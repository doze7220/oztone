#### 【不具合の調査】
現在発生している3つの不具合についてソースコードを調査し、原因と修正方針を特定すること。
**※ 本タスクでは絶対にコードの修正（ファイル書き換え）を行わないこと。**
特定した原因と方針をまとめ、後続の開発AIへ渡すための「引き継ぎ資料」として上記の作業レポートを出力すること。

#### 【発生している不具合】
1. **トラックNoが消えた**: シークバー右上などの「TRACK XXX/XXX」といったトラック数表記が描画されていない。
2. **位置ズレ**: 音量コントロールのパーセント表示や、シークバーの再生時間テキストの描画位置がずれている。
3. **プレイリストがスライドインしない**: 画面右端をホバーしてもプレイリストがスライドインしてこない（裏で曲ジャンプ自体は機能しているため、Window側のホバー判定とLayoutCalculatorの領域計算は正常）。

#### 【調査の絶対ルール (Constraints)】
* **アーキテクチャの維持**: 新しいクラスやメソッドを増やしたり、Rendererの責務（Facadeであること）を壊すような方針は立てないこと。
* **Widget間の独立性**: 情報が足りない場合、Widget同士を直接参照させるのではなく、必ず `WidgetContext` を経由して Application → Renderer → Widget の順でデータを渡す方針とすること。

4. **不具合3の原因と修正手順** (対象ファイルと具体的な修正コード案)
