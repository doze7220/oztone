【今回の目的：Phase 2 - Step 1】 画像、UI、そして将来的な3Dポリゴンのビジュアライザ描画を見据えた土台として、「Direct3D 11」と「Direct2D」を連携させた描画エンジンクラス（Renderer）を導入します。今回は画像の読み込み等は行わず、ウィンドウのクライアント領域を単色（黒など）で塗りつぶす処理のみを実装します。
【実装要件】
src/Renderer.h, src/Renderer.cpp を新規作成してください。
将来的な3D描画と2D描画の共存のため、以下のハイブリッド初期化処理を実装してください。
D3D11CreateDeviceAndSwapChain を用いて、Direct3D 11のデバイスとスワップチェインを作成する。
取得したDXGIサーフェスを利用して、Direct2D (D2D1) のデバイスコンテキストおよびレンダーターゲット（ビットマップ）を作成し、D3D11上に2Dを描画できる環境を構築する。
毎フレーム（または WM_PAINT 発生時）、画面全体を単色（例：D2D1::ColorF::Black）でクリアする Render() メソッドを実装してください。今回はD2D側の Clear 呼び出しのみで構いません。
Application クラスや Window クラスを修正し、Renderer の初期化と描画の呼び出しを適切に組み込んでください。
CMakeLists.txt に d3d11.lib, d2d1.lib, dxgi.lib 等の必要なライブラリのリンク設定を追記すること。
プロジェクト憲法の「WndProcの肥大化禁止」「責務分離」に従い、Windowクラスのメッセージプロシージャ内に直接DirectXの初期化や描画処理を書くことは絶対に避け、すべて Renderer クラスへカプセル化してください。
