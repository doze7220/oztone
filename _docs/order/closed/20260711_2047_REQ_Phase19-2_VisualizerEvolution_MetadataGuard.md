###### 【実装要件】
プレイリスト空の状態から1曲目を追加・再生した際、`UpdateTrackMetadataIfNeeded` によるタグ情報の自己修復処理が走り、TSVキャッシュに波形解析の初期値（`peakAmplitude = 1.0`, `maxFrequency = 2047` 等）が上書き保存されてしまう非同期のすれ違いバグを修正する。

*   **対象ファイル: `src/PlaylistManager.cpp` （またはメタデータ更新処理を行う該当ファイル）**
    *   `PlaylistManager::UpdateMetadata` などのメタデータ上書き更新処理において、引数として渡された新しい `TrackMetadata` で古いデータを上書きする際、**`peakAmplitude` と `maxFrequency`（または `maxHz` 等のスキャン結果パラメータ）については絶対に上書きせず、既存のプレイリストデータが持っている現在の値（裏スレッドでスキャンされた値）をそのまま維持（コピー）するように保護**すること。
    *   これにより、TagLibから取得したタグ情報（曲名・アーティスト名等）だけが安全に更新され、FFTの事前スキャン結果が初期化されてTSVに誤保存されるバグを防ぐことができる。
