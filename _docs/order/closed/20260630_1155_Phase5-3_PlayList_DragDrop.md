現在のプロジェクトをベースに、次の実装を開始します。

**【今回の目的：Phase 5 - Step 3】**
ドラッグ＆ドロップ（D&D）によるプレイリストへの曲追加機能と、リストのファイルへの自動出力機能を実装し、テスト用のハードコード再生を破棄します。

**【実装要件】**
1. **Window の D&D 受付と領域判定**:
   - `Window::Initialize` にて `DragAcceptFiles(hwnd, TRUE)` を呼び出し、D&Dを有効化してください。
   - `WindowProc` で `WM_DROPFILES` を捕捉し、`DragQueryPoint` でドロップ座標を取得してください。その座標を `IsInLogoRegion(x, y)` に渡し、**「左上のアプリアイコン（ロゴ領域）内へのドロップ」かつ「現在ホバー状態であること」**を条件にファイルを受け入れてください。領域外の場合は無視（`DragFinish`）してください。

2. **堅牢なファイル抽出と検証 (Application / PlaylistManager)**:
   - ドロップされたパスがディレクトリの場合は、`std::filesystem::recursive_directory_iterator` を用いてサブフォルダを含めたすべてのファイルを再帰的に検索してください。
   - 見つかったファイルに対し、以下の2段構えの検証を行ってください。
     A. 拡張子が `.mp3` であること（大文字小文字問わず）。
     B. **マジックナンバー検証**: ファイルをバイナリモードで開き、先頭数バイトを読んで、ID3v2タグのヘッダ (`ID3`) またはMP3の同期ワード (`0xFF` 等) で始まっているかを簡易判定し、不正なファイルを弾いてください。

3. **PlaylistManager の拡張 (重複排除と一括自動保存)**:
   - キューに追加する際、すでに同じファイルパスが存在する場合は追加をスキップ（重複排除）してください。
   - D&Dによる複数ファイル・フォルダの解析および `PlaylistManager` への追加処理が **すべて完了した最後に1回だけ**、現在のプレイリストの全パスを `ConfigManager` が指し示すディレクトリ（無ければ exe 同階層の `playlists/current_playlist.txt` 等）へテキストファイルとして自動出力（上書き保存）してください。

4. **再生開始の制御とテストコードの破棄**:
   - `Application::Initialize` に記述していた `test1.mp3` 等のハードコード追加処理、および初回再生開始処理を **完全に削除** してください。
   - ドロップによってファイルが追加された際、**「もし現在 PlaylistManager のキューが新規（空）から1曲以上になった（再生が止まっている）場合」は即座に再生を開始**し、「すでに別の曲が再生中の場合」は再生を止めずにキューへ追加するのみとしてください。
