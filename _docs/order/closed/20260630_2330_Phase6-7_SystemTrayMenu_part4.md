**【今回の目的】**
Phase 6 - Step 7 - part 1 で実装した「Z-Order: 最背面 (Bottom)」が、クリック時などに手前に出てきてしまい「常に最背面」を維持できない（Win32 APIの仕様）問題を解消します。

**【実装要件】**
1. **WindowProc への処理追加**:
   - `Window.cpp` の `WindowProc` 内にて、新たに `WM_WINDOWPOSCHANGING` メッセージを捕捉してください。
   - `lParam` を `WINDOWPOS*` にキャストし、`ConfigManager` の `GetZOrder()` が `2` (最背面) に設定されているかどうかを判定してください。
   - `GetZOrder() == 2` の場合、`WINDOWPOS` 構造体の `hwndInsertAfter` を `HWND_BOTTOM` に上書きしてください。これにより、ウィンドウが手前へ移動しようとする挙動を強制的にキャンセルし、常に最背面を維持させます。
   - ※既に `WM_WINDOWPOSCHANGING` を別の用途で処理している場合は、その中に追記してください。

2. **WM_ACTIVATE などのフォーカス奪取防止 (任意/推奨)**:
   - 最背面モード時にウィンドウがアクティブになることでタスクバー等の挙動がおかしくなるのを防ぐため、必要に応じてフォーカス制御の微調整を行ってください（`WM_WINDOWPOSCHANGING` の対応だけでも基本要件は満たせます）。
