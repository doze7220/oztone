1. 以下の【実装要件】を満たす実装計画書を作成する。
2. 作業ログ（_docs/logs/YYMMDD_HHMM_Phase8-8_Playlist_Interaction.md）を新規作成し、タスクリストを末尾に記載する。
2. タスクリストの作成が終わったら、絶対にコードの修正は行わずに、まずはチャットで「タスクリストの作成が完了しました。タスク1を実行してよいか指示をお願いします」と報告し、私の承認を待つこと。

【実装要件】
1. PlaylistManager の拡張
対象: src/PlaylistManager.h, src/PlaylistManager.cpp
- 指定したシャッフルインデックスへ直接ジャンプするメソッド `JumpToIndex(size_t index)` を実装してください。
- 内部のインデックス（現在の再生位置）を更新し、次回のキュー等の整合性を保つこと。

2. スクロール状態の管理と描画への反映
対象: src/Renderer.h, src/Renderer.cpp, src/Application.cpp
- プレイリストの手動スクロールオフセットを保持する変数を追加してください（例: `m_playlistManualScrollY`）。
- プレイリストが閉じられた際（`isPlaylistHovered` が false になった瞬間）には、この手動オフセットを 0 にリセットし、次に開いた時は自動的に「現在再生中の曲」が中央に来る仕様を維持してください。
- `Renderer::Render` にて、プレイリスト項目を描画する際のY座標計算にこの手動オフセットを加算し、一番上および一番下までスクロールした際に余白ができすぎないように `std::clamp` 等で制限をかけてください。

3. Window の入力イベント処理（スクロールとクリック）
対象: src/Window.cpp, src/Application.cpp (コールバック経由)
- `WM_MOUSEWHEEL`: プレイリストのホバー判定領域内にいる場合、このメッセージをプレイリストのスクロール処理として扱い、スクロールオフセット変数を更新してください。
- `WM_LBUTTONDOWN`: プレイリストホバー判定内にいる場合、クリックされたY座標、現在のスクロールオフセット、および ConfigManager の `PlaylistItemOffsetY` などのレイアウト値を用いて「シャッフルリストの何番目の曲がクリックされたか」を計算してください。
- 算出されたインデックスを `PlaylistManager::JumpToIndex` に渡し、対象の曲の再生（AudioPlayer::Play）を開始する処理を実行してください。
