**【今回の目的：Phase 4 - Step 4】**
TagLibを導入してMP3ファイルからメタデータ（曲名、アーティスト名、埋め込みアルバムアート）を取得し、UIへ反映させます。同時に、左下アルバムアートの描画ロジックを「アスペクト比維持＆中央センタリング」に改良します。

**【実装要件】**
1. **ライブラリの導入とクラスの新設 (TagManager)**:
   - `TagLib` をプロジェクトに導入し、CMakeLists.txt を適切に更新してください（FetchContent等の利用を推奨）。
   - タグ解析の責務を担う `TagManager` (または `TagParser`) クラスを新規作成してください。
   - ファイルパスを受け取り、TagLibを用いて「曲名 (wstring)」「アーティスト名 (wstring)」「アルバムアートのバイナリデータ (std::vector<uint8_t> 等)」を抽出・保持するメソッドを実装してください。

2. **Renderer の拡張 (メモリ画像デコードとテキスト更新)**:
   - `WIC` を用いて、渡されたバイナリデータから `ID2D1Bitmap` を生成するメソッド（例: `LoadBitmapFromMemory`）を追加してください（`IWICStream::InitializeFromMemory` 等を活用）。
   - 現在固定で描画している「曲名」と「アーティスト名」を、動的な文字列で描画できるようにメンバ変数や Render メソッドの引数を拡張してください。

3. **Renderer の描画ロジック改良 (左下アートのセンタリング)**:
   - 左下アルバムアートの描画処理を以下のように修正してください。
   - 対象ビットマップの元のピクセルサイズ（縦横幅）を取得し、アスペクト比（縦横比）を計算してください。
   - ConfigManager から取得した `ArtSize`（正方形の描画領域）に対して、アスペクト比を維持したまま内接するように「描画する幅と高さ」を計算してください。
   - 計算した幅と高さを用いて、`ArtSize x ArtSize` の領域の**「完全な中央（縦横ともにセンタリング）」**となる Destination Rectangle (描画先矩形) を算出し、`DrawBitmap` で描画してください。

4. **Application からのデータ結合**:
   - `Application::Initialize` にて、`test.mp3` の再生直前に `TagManager` にパスを渡し、情報を抽出してください。
   - 抽出した曲名・アーティスト名・画像データを `Renderer` に設定してください（取得できない場合は "Unknown" 等のテキストや既存のプレースホルダー画像へフォールバックさせてください）。


**【追加処理】**
【AIへの追加指示（メタデータ未定義時のフォールバック処理とアルファ設定）】
先ほど実装したTagLibによるメタデータ取得処理に対して、タグ情報が未定義だった場合のフォールバック（デフォルト）の挙動を以下の通り追加実装してください。

1. **ConfigManager の拡張（黒い板のアルファ値）**:
   - `[Layout_NowPlaying]` セクションに `FallbackArtOpacity` (float型) を追加し、読み書きできるようにしてください。デフォルト値は `0.5f` （半透明の黒）とします。

2. **曲タイトルのフォールバック**:
   - TagLibから曲名が取得できなかった場合、`std::filesystem::path::filename()` 等を用いて、再生するファイルのパスから「拡張子込みのファイル名（例: test.mp3）」を抽出し、それを曲名として `Renderer::SetTrackInfo` へ渡してください。

3. **アーティスト名のフォールバック**:
   - TagLibからアーティスト名が取得できなかった場合、文字列 `L"---"` をアーティスト名として `Renderer::SetTrackInfo` へ渡してください。

4. **アルバムアートのフォールバック（背景と左下で描画を分岐）**:
   - `Renderer` クラスにおいて、現在「タグから抽出された正規のアルバムアートがセットされているか」を判定する状態（フラグ等）を管理してください。
   - タグに画像が存在しない場合、**背景**には引き続きプレースホルダー画像（IDI_PLACEHOLDER_ART）をフィット・トリミングして描画してください。
   - タグに画像が存在しない場合、**左下のアルバムアート領域**にはプレースホルダー画像を描画せず、Direct2Dの単色ブラシを用いて `FillRectangle` で「黒い正方形の板」を描画するようにロジックを分岐させてください。
   - この「黒い板」を描画する際、ブラシの色は `D2D1::ColorF(0.0f, 0.0f, 0.0f, ConfigManagerから取得したFallbackArtOpacity)` とし、INI設定でアルファ値を調整できるようにしてください。
