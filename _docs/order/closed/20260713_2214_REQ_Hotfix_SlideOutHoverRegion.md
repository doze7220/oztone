#### 【実装要件】
プレイリストやロゴ拡張メニューが「スライドアウト（格納アニメーション）」している最中に、拡張された広いホバー判定領域にマウスが触れて意図せず再展開してしまう不具合を修正する。

*   **要件1: 展開状態フラグ（拡張フラグ）の厳格化**
    *   `src/Renderer.cpp` の `UpdateAnimation` メソッド、または `src/Application_Render.cpp` における「Windowクラスへ渡すUI展開状態フラグ (`isPlaylistExpanded`, `isLogoMenuExpanded` など)」の算出ロジックを調査する。
    *   現在、スライドアニメーションの進行度（SlideX > 0 等）に依存してフラグが `true` になっている場合、スライドアウト中も広いホバー領域が維持されてしまう。
    *   フラグが `true` になる条件を、「実際にマウスがホバーしている (`isHovered` == true)」または「離脱ディレイタイマーが残っている (`m_leaveTimer > 0.0f`)」状態のみに限定（厳格化）する。
    *   これにより、ディレイが切れて格納アニメーションが始まった瞬間に `isPlaylistExpanded` などのフラグが `false` となり、Window側 (`IsInPlaylistRegion` 等) のホバー判定領域が元の狭い幅に戻るように修正すること。

#### 【絶対遵守ルール (Constraints)】
*   **アニメーションの維持**: 格納アニメーション（スライドアウトする視覚的挙動）自体は途切れることなく最後まで描画されるようにすること。判定領域のフラグのみを修正対象とする。
