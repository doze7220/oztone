【AIへの指示（Phase 8 - Step 3 : 対応ファイルフォーマットの拡張）】
Phase 8のブラッシュアップとして、ドラッグ＆ドロップで受け付ける対応ファイルフォーマットを拡張します。
`miniaudio` が標準対応しているフォーマット（公式対応）と、OSのMedia Foundation等へのフォールバックを期待するフォーマット（実験的対応）の2段階で実装を行います。

以下の要件に従って `src/Application.cpp` の `OnFilesDropped` メソッド内の判定ロジックを修正してください。

**【実装要件】**
1. **許可拡張子の追加 (Application::OnFilesDropped)**:
   - 再帰探索およびファイル判定時の拡張子チェックに、公式対応として `.flac`, `.wav`, `.ogg` を追加してください（大文字・小文字を区別しないこと）。
   - さらに「実験的（仮実装）な対応」として `.mp4`, `.m4a` を追加してください。
   - **[重要]** `.mp4`, `.m4a` に関する処理（拡張子判定や分岐）は、将来的に容易に削除・改変できるよう、必ず `// [EXPERIMENTAL] MP4/M4A Support` のような明確なコメントブロックで囲んで隔離してください。

2. **マジックナンバー（ファイルヘッダ）検証の拡張**:
   - 拡張子だけでなく、ファイルの中身が正しいフォーマットであるかを判定する現在のマジックナンバー検証ロジックを拡張してください。
   - 以下のシグネチャを判定できるように分岐を追加してください。
     - MP3 (既存): `ID3` または `0xFF`
     - FLAC: 先頭4バイトが `fLaC`
     - OGG: 先頭4バイトが `OggS`
     - WAV: 先頭4バイトが `RIFF`
   - **実験枠 (`.mp4`, `.m4a`) の扱い**: 動画コンテナ構造は検証が複雑なため、これらの実験的拡張子を持つファイルについては**マジックナンバー検証をスキップして常に通過させる（無条件でプレイリスト追加を許可する）**処理にしてください。このスキップ処理も `[EXPERIMENTAL]` コメントブロック内に含めてください。

**【事後処理】**
ビルドを行い、FLAC, WAV, OGG ファイルがD&Dでプレイリストに追加され正常に再生・タグ読み込みされること。また、MP4 ファイルをドロップした際にエラーで弾かれず、映像は出ないものの「音声トラック」として音が鳴るかどうか（miniaudio の OS フォールバック機能が働くか）を確認して報告してください。
