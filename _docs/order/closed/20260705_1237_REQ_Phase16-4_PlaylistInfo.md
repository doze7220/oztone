#### 【実装要件】
プレイリストのUIにおいて、曲一覧モードとプレイリスト一覧モードでそれぞれ最適な情報表示を行えるように改修する。

* **要件1: 曲一覧モード時のツールバー表示 (PlaylistWidget)**
  * 曲一覧モード時（`m_isPlaylistListViewMode == false`）、非ホバー時のツールバー下段の「説明テキスト領域」には、シンプルに現在の「プレイリスト名（拡張子抜き）」のみを1行で表示する。
  * ホバー時は従来通りボタンの説明テキストで上書きする。
* **要件2: プレイリスト一覧のメタデータ取得・キャッシュ (Application / ConfigManager 等)**
  * プレイリスト一覧モードの描画に必要な「曲数」と「総時間」を取得するため、`.ozl`（または `.lst`）ファイルの軽量な解析・キャッシュ機能を実装する。
  * Phase 10-4で実装されたTSV形式（`filepath \t title \t artist \t timeString`）を読み込む。
  * 行数を「曲数」とする。
  * `timeString` をパース・合算して「総時間」を算出するが、全て解析が終わっておらず「ファイルパスだけの行（要素が足りない未解析の行）」が見つかった場合は、そのファイルの合算をスキップし、プレイリスト全体の総時間テキストをストイックに `---` とする。
  * ※毎フレームのファイルI/Oを防ぐため、必ずキャッシュを作成し、`WidgetContext` を経由して描画層へ渡すアーキテクチャとすること。
* **要件3: プレイリスト一覧モード時の2行スタイル描画 (PlaylistWidget)**
  * プレイリスト一覧モード時（`m_isPlaylistListViewMode == true`）、リストに並ぶ各プレイリスト項目の描画を、曲アイテムと同じ2行スタイルに改修する。
  * **上段**: 左寄せで「プレイリスト名」
  * **下段**: 左寄せで「曲数テキスト（例: 150 track）」、右寄せで「総時間テキスト（例: 1:12:45 または ---）」を描画する。
  * 既存のリソース（`m_playlistTitleTextFormat`, `m_playlistArtistTextFormat`, `m_playlistTimeTextFormat` 等）を再利用すること。

#### 【実装要件】
Phase 16-4の改修により発生した以下の2つの描画バグを修正する。裏側のクリック判定等は正常に動作しているため、描画層の分岐ロジックのみを改修すること。

* **要件1: プレイリスト一覧モードの描画データの完全分離 (PlaylistWidget)**
  * 現在、プレイリスト一覧モードのツールバーは正しく描画され、クリック判定も正常（別のプレイリストが開かれる）だが、リスト部分の描画内容が「曲一覧」になってしまうバグが発生している。
  * `PlaylistWidget::Draw` のリストアイテム描画ループにおいて、`ctx.isPlaylistListViewMode == true` の場合は、**ループの回数（上限）を `ctx.totalTracks` ではなく、プレイリストファイルの総数にする**こと。
  * 同時に、描画するテキスト（タイトル、曲数、時間等）も、現在聴いている曲のリストではなく、要件で実装した「プレイリストのキャッシュメタデータ（PlaylistSummary等）」から取得するように、ループ内のデータ参照ロジックを完全に分岐・修正すること。
* **要件2: 曲一覧モードのツールバー非ホバー時のプレイリスト名表示 (PlaylistWidget)**
  * 曲一覧モード（`ctx.isPlaylistListViewMode == false`）において、ツールバー領域からマウスが外れた際（非ホバー時）、プレイリスト名が表示されないバグを修正する。
  * `PlaylistWidget::Draw` 内のツールバーテキスト決定ロジックを見直し、非ホバー時（`ctx.playlistToolbarHoveredIndex == -1`）には、`ConfigManager::GetDefaultPlaylistPath()` 等から取得した現在のプレイリスト名（拡張子抜き）を1行で描画する処理を確実に実行させること。
