#### 【実装要件】
OZtoneの「極上の手触り」をさらに強化するため、各UI要素をクリックした際の視覚的なリアクション（アニメーション）を実装する。

*   **要件1: ワンショット・クリック検知機構の導入**
    *   現在、クリック入力は `Window` から直接バックエンド（Application等）へ伝達されているが、描画側（Widget）でアニメーションを開始させるためのトリガーが必要である。
    *   `Window` 内でクリック（`WM_LBUTTONDOWN`）が発生した際に、該当する対象（プレイリストのインデックス、ロゴメニューのID、プレイコントロールのボタン種別など）を1フレームだけ保持し、`Application::ForceRender` 経由で `WidgetContext` へ「今クリックされた（ワンショット）」という情報を伝達する仕組みを構築する。

*   **要件2: アイコン波紋（残像広がり）エフェクト (AppLogoWidget, LogoMenuWidget, PlaybackControlsWidget)**
    *   各Widgetに波紋アニメーションの進行度（0.0f〜1.0f等）を管理する変数を追加し、`UpdateAnimation` で更新する。
    *   描画時、通常のアイコン/画像を描画した直後に、波紋用の進行度を用いて `D2D1::Matrix3x2F::Scale` で中心を基準に1.5倍程度に拡大させつつ、不透明度を 0.5f から 0.0f へフェードアウトさせて「アイコン自身の形の残像が広がる」エフェクトを重ねて描画する。

*   **要件3: プレイリストのレーダー走査線＆再生色染めエフェクト (PlaylistWidget)**
    *   `PlaylistWidget` のアイテムにクリックアニメーションの進行度（X座標または0.0f〜1.0f）を管理する変数を追加。
    *   曲がクリックされた瞬間から、アイテムの左端から右端へ向かって走査線（X座標）が進むアニメーションを `UpdateAnimation` で処理する。
    *   描画時、以下の手順で表現する：
        1. 通常色でテキストを描画。
        2. 走査線位置から左に向かって、線形グラデーションブラシ（右端がPlayingItemColor、左端が透明）を用いて「レーダーの尾」の矩形を描画。
        3. 走査線位置より「左側」の領域だけを `PushAxisAlignedClip` で切り抜き、そのクリップ領域の中に PlayingItemColor の単色ブラシでテキストを再描画（上書き）する。これにより「線が通り過ぎた部分から再生色に染まる」サイバーな演出を実現する。

#### 【追加・変更要件（計画書からの上書き）】
*   **タスク4: プレイリストのレーダー走査線エフェクトのトリガー変更**
    *   **変更前**: 「アイテムがクリックされた瞬間」にアニメーションを開始する。
    *   **変更後**: 「**再生中の曲（`currentTrackIndex`）が切り替わった瞬間**」にアニメーションを開始する。
    *   **実装方針**: `PlaylistWidget` の `UpdateAnimation` 内で、前フレームの `currentTrackIndex` と `ctx.currentTrackIndex` を比較し、変化があった場合に走査線アニメーションの進行度（`m_radarProgress`等）をリセットしてアニメーションを開始させること。
    *   これにより、クリック操作だけでなく、曲の自動進行やスキップボタン、ホットキー操作など、あらゆる手段で曲が切り替わった際に必ずレーダーエフェクトが発動するようになる。
*   **補足**: タスク1の「ワンショット・クリック検知機構」は、アイコンや再生コントロールの「波紋エフェクト（タスク2, 3）」のために計画通り実装すること。ただし、上記の変更によりプレイリストの演出にはこの検知機構は使用しない。
