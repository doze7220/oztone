###### 【実装要件】
*  `src/Application_FileDrop.cpp` の `OnFilesDropped` メソッド内を改修する。
*  現在の実装では、ファイルのドロップ追加時（`!isShiftPressed`）において、バックグラウンドの解析スレッドを起床させるエンキュー処理（未解析トラックのキュー追加および `m_parseCV.notify_one()` 等）が、メインスレッドの即時再生（`PlayCurrentTrack()`）よりも先に行われている。これが原因でミリ秒単位のファイルロック競合が発生し、再生失敗による意図しない曲スキップ（フェイルセーフの誤発動）を引き起こしている。
*  このバグを解消するため、**「解析スレッドへのエンキューおよび起床処理全体」を、即時再生（`PlayCurrentTrack()`）呼び出しの「後」へ移動させる。**
*  これにより、メインスレッドが優先してファイルをオープン（再生開始）できるように順序を適正化する。
