【AIへの指示（Phase 8 - Step 1 : 忍者UI・再生コントロールとINIレイアウト連携）】
Phase 8のブラッシュアップとして、OZtoneのコンセプトである「ストイックな忍者UI」に基づいた再生コントロール機能を実装します。
UIレイアウトはハードコードせず、INIファイルから「画面中央・下部」を基点としたオフセットで定義できるようにアーキテクチャを拡張してください。

以下の要件に従って関連ファイルを修正してください。

**【実装要件】**
1. **ConfigManager の拡張 (INIレイアウト定義)**:
   - 新たに `[Layout_PlaybackControls]` セクションを追加し、デフォルトINIテンプレート（`DEFAULT_INI_CONTENT`）に追記してください。
   - 以下のパラメータの読み込みとゲッターを実装してください。
     - `BaseBottomOffset` (ウィンドウ下端からのY座標基点, 例: 40.0f)
     - `CenterOffsetX` (ウィンドウ中央からのX座標ズレ, 例: 0.0f)
     - `ButtonSpacing` (各ボタン間の距離, 例: 40.0f)
     - `ButtonSize` (ボタンの基本サイズ, 例: 20.0f)

2. **再生コントロールアイコンの描画とホバーアニメーション (Renderer)**:
   - コントロール領域周辺にマウスが入った時だけフワッと表示させるため、不透明度（アルファ値）を毎フレーム加算/減算するフェードアニメーションを実装してください。
   - 描画位置は、`ConfigManager` から取得した値を用い、`X = (論理幅 / 2.0f) + CenterOffsetX`、`Y = (論理高さ - BaseBottomOffset)` を中央の基点とします。
   - この基点から `ButtonSpacing` の間隔を空けて、左から「前の曲 (⏮)」「再生/一時停止 (▶ / ⏸)」「次の曲 (⏭)」の3つのボタンを描画してください。
   - 外部画像は使わず、Direct2Dのパス描画（`ID2D1PathGeometry` 等）を用いて、白のベクターアイコンとして描画してください。再生状態 (`AudioPlayer::IsPlaying()`) に応じて中央のアイコンを切り替えてください。

3. **クリック検知とアクション実行 (Window / Application)**:
   - `WindowProc` での `WM_LBUTTONDOWN` 時、マウスの論理座標が上記の計算式で求めた各ボタンの矩形領域内（`ButtonSize` ベース）にあるかを判定する処理を追加してください（領域判定時に `ConfigManager` の値を使用すること）。
   - 各ボタンがクリックされたら、Phase 6で実装したメディアキー対応と同じロジック（`TogglePlayPause()`, `Previous()`, `Advance()` + 再読み込み）を呼び出して再生状態を制御できるようにしてください。

**【事後処理】**
ビルドを行い、普段はシークバーしか見えないが、マウスを画面下部中央に持っていくと再生コントロールがフワッと浮かび上がり、クリックで曲の制御が行えること。また、INIファイルを編集することでボタンの位置や間隔が動的に変わることを確認して報告してください。
