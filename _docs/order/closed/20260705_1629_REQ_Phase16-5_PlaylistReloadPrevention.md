#### 【実装要件】
プレイリスト一覧モードから「現在再生中のプレイリスト」をダブルクリックして再度開こうとした際、音楽が停止してプレイリストが最初から再読み込みされてしまう挙動を改善する。

* **要件1: 同一プレイリスト判定と再ロードの防止 (Application)**
  * プレイリスト一覧モード時のダブルクリックコールバック、または `Application::SwitchPlaylist` の先頭において、要求されたファイルパスが「現在読み込まれているプレイリスト（`m_config->GetDefaultPlaylistPath()` 等）」と一致するかどうかを判定するロジックを追加する（パス比較の際は `std::filesystem::path` 等を用いて等価性を確認すること）。
  * パスが一致する場合（同一プレイリストの場合）は、内部キューのクリア・再ロード・再生の初期化といった一連の処理をすべてスキップし、再生を維持したまま単にプレイリスト一覧モードを終了（`m_isPlaylistListViewMode = false;`）して曲一覧モードへ戻る処理のみを行う。
  * パスが異なる場合は、従来通り `SwitchPlaylist` による別プレイリストへの切り替え処理を実行する。
