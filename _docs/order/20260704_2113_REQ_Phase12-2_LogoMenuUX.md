#### 【実装要件】
本タスクの目的は、前回実装したロゴ拡張メニューの操作性・視認性の課題を解消するため、個別のホバー演出（画像・グロー）、説明テキストの動的切り替え、プレイリストとの干渉排除、およびメニュー展開中のロゴホバー状態維持を実装することである。

* **要件1: プレイリストホバー判定の干渉排除 (Window)**
  * プレイリストが左配置設定の場合、ロゴ拡張メニューの操作中に誤ってプレイリストが展開して操作性が著しく低下する問題がある。
  * `Window::IsInPlaylistRegion` にて、プレイリストの左右配置に関わらず、**「ロゴアイコン（および展開メニュー）が存在する高さ（Y座標がロゴの高さ以下の領域）」** をプレイリストのホバー判定から完全に除外する。
* **要件2: 個別アイコンのホバー判定と視覚フィードバック (Window / Renderer / Widget)**
  * `Window` 側で、展開中のメニューの「どのアイコン上にマウスカーソルがあるか」を判定する仕組み（`GetLogoMenuButtonAt` 等）を実装し、そのインデックスを状態として管理・伝達する。
  * `LogoMenuWidget` の描画にて、ホバー中のアイコンの背景に、新規画像 `app_logo_back.png` をアイコンと同じサイズ・不透明度0.6で描画する（フェード等のアニメーション値を持たせるとさらに良い）。画像の読み込みは既存の `LoadBitmapResource` 等を使用する。
  * さらにホバー中のアイコン自体にグロー効果（D2D1Shadow の色変更や、太いパス描画などを活用）を付与し、タップ可能なアフォーダンスを明示する。
* **要件3: アイコン説明テキストの動的表示 (Widget)**
  * 従来のタイピング演出を廃止し、ホバー中のアイコンに対応した以下の機能説明テキストを上部の領域に描画するよう改修する。
    * アプリ終了 (❌) -> `OZtoneの終了`
    * 再生中の曲削除 (🗑️) -> `再生中の曲をプレイリストから削除する`
    * プレイリスト固定 (📜) -> `プレイリスト固定表示`
    * ビジュアライザ切替 (📽️) -> `ビジュアライザ表示切り替え`
    * シャッフル (🔀) -> `シャッフル再生ON/OFF`
  * どのアイコンにもホバーしていない状態でのデフォルトテキスト（空白とするか、"OZtone Menu" 等とするか）も美しく設計に組み込むこと。
* **要件4: メニュー操作中のロゴホバー状態維持 (Widget)**
  * ロゴメニューが展開・表示されている間（`isLogoMenuHovered` が true の間）は、アプリアイコン（OZロゴ）自身もホバー状態の画像（`app_logo_hover.png`等）で描画され続けるように `AppLogoWidget` の描画ロジックを修正する。

#### 【絶対遵守ルール (Constraints)】
* **責務分離の厳守**: マウスカーソルがどのアイテム上にあるかの判定は必ず `Window` で行い、状態の伝達は `WidgetContext` 等を介して行うこと。テキストや画像の描画は `LogoMenuWidget` および `AppLogoWidget` 内にカプセル化すること。
