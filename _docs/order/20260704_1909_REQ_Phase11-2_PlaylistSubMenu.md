#### 【実装要件】
本タスクの目的は、システムトレイメニューを拡張して「プレイリスト」というサブメニュー（ネスト化）を作成し、展開するたびにディレクトリ内のファイルを自動スキャンして一覧表示し、クリックでプレイリストを切り替えられるようにすることである。

* **要件1: プレイリストファイルの自動スキャン機構**
  * `ConfigManager` または `Application` にて、現在の `DefaultPlaylistPath` の存在するディレクトリ（親ディレクトリがない場合はカレントディレクトリ）をスキャンする処理を実装する。
  * 拡張子が `.ozl` または `.lst` のファイルのみを抽出し、ファイル名のリスト（パスまたは表示名）を取得できるようにする。
* **要件2: システムトレイメニューの動的ネスト化 (Window)**
  * `WM_TRAYICON` における右クリックメニュー構築処理を改修し、Win32 APIの `CreatePopupMenu()` 等を用いて「プレイリスト (Playlists)」というサブメニューを作成する。
  * サブメニュー展開時（メニュー構築時）に要件1のスキャン処理を呼び出し、見つかったファイル名を動的にメニュー項目として追加する（拡張子を除いたファイル名を表示名とするとより美しい）。
  * 現在読み込まれているプレイリスト（`ConfigManager` の `DefaultPlaylistPath` と一致するもの）にはチェックマーク（`MF_CHECKED`）を付ける。
  * 動的に追加するメニュー項目のコマンドIDは、既存のIDと衝突しないよう予約領域（例: `ID_TRAY_PLAYLIST_START` = 2000 から 2100 まで等）を定義して安全に割り当てること。
* **要件3: プレイリストの動的切り替え (Application / PlaylistManager)**
  * ユーザーがサブメニューから任意のプレイリストを選択（クリック）した際、動的IDから対象のファイルパスを逆算する。
  * `Application::SwitchPlaylist(const std::wstring& filepath)` のような切り替え専用メソッドを新設する。
  * 切り替え処理では、`ConfigManager::SetDefaultPlaylistPath` で新しいパスをINIに保存し、既存の `ClearPlaylist()` 相当の処理でキューと再生をリセットした後、`PlaylistManager::LoadFromFile` で新リストをロードし、要素があれば最初の曲の再生および次曲先読みを開始する。

#### 【絶対遵守ルール (Constraints)】
* **メモリリークの防止**: 動的メニュー（`CreatePopupMenu`）の作成と破棄（`DestroyMenu`）が、Win32の仕様に沿って正しくメモリリークなく行われるように設計すること。
* **スコープの厳守**: 今回はプレイリストの自動認識と切り替えのみをターゲットとする。再生中の曲を削除する機能などにはまだ手を出さないこと。
