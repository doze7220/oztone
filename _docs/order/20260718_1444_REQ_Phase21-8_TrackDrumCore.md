#### 【実装要件】
曲の切り替え時（メディアキー等による Next / Prev 時）に、物理的な「質量」と「慣性」を持ったドラムが回転するトランジションUI（トラックドラム）のコアエンジンを実装する。本機能はINIで有効/無効を切り替え可能とし、無効時（従来モード）はアニメーションを行わず即時切り替えを保証する。
※マウスホイール連動やUIのホバーアフォーダンス等は後続フェーズ(Phase 21-9)で実装するため、本フェーズでは**「描画とバケツリレーの物理演算」**のみにスコープを限定する。

*   **要件1: ConfigManagerの拡張 (`[TrackDrum]` セクション)**
    *   INIファイルに `[TrackDrum]` セクションを新設し、以下のパラメータを管理する。
    *   `EnableTrackDrum` (デフォルト: 1): 機能の有効/無効トグル。0の場合は従来通りの即時切り替えとする。
    *   `MaxDuration` (デフォルト: 3.0): 最長アニメーション時間（秒）。
    *   `MinSpeed` / `MaxSpeed` (デフォルト: 10.0 / 60.0): 動的変化する最低・最高回転速度（スロット/秒）。
    *   `Acceleration` / `Deceleration` (デフォルト: 20.0 / 15.0): ドラムの初速加速度と停止減速度。

*   **要件2: トラックドラムの描画とバケツリレー管理 (Renderer / Widget_TrackInfo 等)**
    *   描画オブジェクトは「消えゆく曲（OLD）」と「現れる曲（NOW）」の2つだけでバケツリレーを行う。
    *   `UpdateAnimation` にて、加速度・減速度を用いて速度をS字カーブで計算し、インデックスを更新する。
    *   遠距離ジャンプ（シャッフル時やプレイリストからの直接ジャンプ等）が入力された場合は、現在の速度と `MaxDuration` から最大表示可能数を割り出し、間のインデックスを間引き（スキップ）して描画を追いつかせる。
    *   超高速回転中は重い画像デコードを行わず、半透明のガラス板（`FallbackArtOpacity` の流用など）を描画。目標地点に着地した瞬間に正規アートの非同期ロードを開始し、完了次第クロスフェードで出現させる。

*   **要件3: シャッフル時および既存操作の安全設計**
    *   静止時にシャッフルが切り替わった場合は表示を動かさず裏のリストのみ更新。アニメーション中の場合はOLD/NOWをそれぞれ着地させてから新リストを引き込む。
