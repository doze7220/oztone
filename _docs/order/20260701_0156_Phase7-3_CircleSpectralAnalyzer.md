【AIへの指示（Phase 7 - Step 5 : 複数ビジュアライザの共存と円形パーティクル実装 改訂版）】
現在の PrismBeatVisualizer に加え、新たな「円形パーティクル・スペアナ」を実装し、それらをシステムトレイメニューから動的に切り替えられるようにアーキテクチャを拡張してください。

以下の要件に従って、関連ファイルを修正してください。

**【実装要件】**
1. **ConfigManager の拡張 (モード管理)**:
   - `m_visualizerMode` メンバ変数を追加し、INIファイルの `[Visualizer]` セクションから設定を読み書きできるようにしてください（0: PrismBeat [デフォルト], 1: CircleParticle）。
   - ゲッター `GetVisualizerMode()` と セッター `SetVisualizerMode()` を実装してください。

2. **システムトレイメニューへの追加 (Window クラス)**:
   - `resource.h` / `Window.h` 等にメニューID（例: `ID_TRAY_VIS_PRISM`, `ID_TRAY_VIS_CIRCLE`）を追加してください。
   - `Window.cpp` のシステムトレイ構築処理（`WM_TRAYICON`）にて、「ビジュアライザの切り替え」メニューを追加し、現在の `VisualizerMode` に応じてラジオボタン（チェックマーク）がつくようにしてください。
   - `WM_COMMAND` にて、選択されたモードを `ConfigManager` に保存し、即座に反映されるようにしてください。

3. **Visualizer クラスの関数分離**:
   - 既存の `Draw` メソッドの中身（直線オシロスコープの描画処理一式）を、新設する `DrawPrismBeat` メソッドへ丸ごと移動（退避）してください。
   - `Visualizer` クラスのメンバに `ConfigManager` へのポインタまたは参照を渡し、大元の `Draw` メソッド内では `m_config->GetVisualizerMode()` の値に応じて `DrawPrismBeat` または後述の `DrawCircleParticle` を呼び出すように分岐させてください。

4. **円形パーティクル・スペアナの実装 (DrawCircleParticle)**:
   - 新規メソッド `DrawCircleParticle` を作成し、以下の要件を満たす描画を実装してください。
   - **円形バー**: 画面中央を原点とする基本半径 `R` に対し、`SetTransform` でキャンバスを回転させながら `FillRectangle` で板を描画し、振幅に応じて内側・外側に膨張させてください。
   - **固有カラー**: 曲名・アーティスト名の結合文字列（TagManagerから取得）を `std::hash` でハッシュ化し、Hue(0-360)へ変換して、その曲固有のテーマカラーで描画してください。
   - **パーティクル放出と3D回転描画**: 前フレームより振幅が小さくなった瞬間に、バーの先端から外へ向かって飛散するパーティクル（四角形の板ポリゴン）を生成してください。
     - 各パーティクルは生成時に「X, Y, Z 全ての成分がランダムな値を持つ、任意の3Dベクトル（回転軸）」を1つ決定し、寿命が尽きるまでその軸を保持します。
     - 描画時は、Direct2Dの2D変換ではなく、その3D軸を中心に板ポリゴンの4頂点を3D空間上で回転計算（ロドリゲスの回転公式等を適用）したのち、2D平面に投影し、`ID2D1PathGeometry` 等を用いて四角形として描画してください。
   - **放射状の光**: 最背面に、円の中心から外周へ向かって伸びる、ゆっくりと回転する半透明の光の筋（ラインまたはポリゴン）を描画してください。

**【事後処理】**
ビルド後、システムトレイメニューからビジュアライザを切り替え、既存の美しい直線ネオン（PrismBeat）と、曲ごとに色が変わり、任意の3D軸で立体的に回転する破片が飛び散る新しい円形ビジュアライザ（CircleParticle）が相互に切り替わり、どちらも正常に動作することを確認して報告してください。
