**【今回の目的：Phase 7 - Step 1】**
メディアプレイヤーの華である「ビジュアライザ」の実装基盤を作ります。まずは円形などの複雑な描画を行う前に、miniaudioから再生中の音声データをキャプチャしてFFT（高速フーリエ変換）解析を行い、抽出した周波数スペクトルをDirect2Dを用いて「シンプルなライン（またはバー）のスペクトルアナライザ」として画面上に描画します。

作業開始前に、タスクリストを作ってください。
今回はステップを踏んで、実装を行います。
下記実装要件が１つ終わるごとに、ユーザーに報告してください。
ビルド・挙動に問題がないことを確認してから、資料を更新し、次ステップへと進みます。

**【実装要件】**
1. **AudioPlayer の拡張 (音声キャプチャとFFT解析)**:
   - `AudioPlayer.h` / `.cpp` にて、`miniaudio` のノードグラフ機能 (`ma_node`) またはエンジンのポストミックスコールバック（`ma_engine_config` の `dataCallback` 等）を用いて、再生中のPCMオーディオデータをリアルタイムにキャプチャする仕組みを実装してください。
   - オーディオスレッドとメインループ（描画スレッド）間のデータ受け渡しとなるため、`std::mutex` を用いてスレッドセーフに波形データをバッファリングしてください。
   - プロジェクト内に外部FFTライブラリは無いため、キャプチャしたデータに対して動作する**軽量でシンプルなC++ネイティブのFFT処理（高速フーリエ変換アルゴリズム）**をクラス内（または無名名前空間）に自作実装し、波形を周波数スペクトルデータ (`std::vector<float>` など) に変換してください。
   - 最新のスペクトルデータをメインループ側から取得するためのメソッド `GetSpectrumData(std::vector<float>& outSpectrum)` を追加してください。

2. **Renderer の拡張 (シンプルなスペアナ描画)**:
   - `Renderer::Render` メソッドの引数に、スペクトルデータ `const std::vector<float>& spectrum` を追加してください（宣言と実装の両方を更新）。
   - 画面の下半分程度の領域を使って、取得したスペクトルデータを元に `ID2D1SolidColorBrush` と `DrawLine` (または `FillRectangle`) を用いて、シンプルなスペクトルアナライザ（周波数の強弱を示すラインまたはバーグラフ）を描画してください。
   - ※今回はデータ連携のテストが目的のため、色やデザインは仮のもので構いません。白や目立つ色の単色ブラシで描画してください。

3. **Application の連携**:
   - `Application.h` / `.cpp` を修正し、`m_renderer.Render` の呼び出し箇所に合わせる形で対応を行ってください。
   - メインループ (`Run`) 内の毎フレームの処理において、`m_audioPlayer.GetSpectrumData(spectrum)` を呼び出して最新のデータを取得し、それを `Render` メソッドに渡すようにロジックを結合してください。

**【事後処理】**
実装完了後、ビルドを行ってください。
音楽を再生した際に、音声に合わせて画面上にシンプルなスペクトルアナライザがピョコピョコとリアルタイムに動くこと（データ取得と描画の連動）を確認できたら、実装レポートを出力してください。
