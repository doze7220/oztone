Phase 21-1: 極上スロット・トランジション＆CD帯UI 仕様書
1. コンセプト・UX
    - 物理メディアの所有感: デジタルなアプリの中に、CDやレコードのようなフィジカル（物理的）な手触りを再現する。
    - アートへの絶対的リスペクト: 画像の上に文字やグラデーションを一切被せず、100%ピュアなアルバムアートを完全な形で見せる。
    - メカニカルな心地よさ: アナログのジュークボックスやCDチェンジャーのように、曲の切り替わり時に物理パネルが「カシャッ」と入れ替わるスロット・アニメーション。

2. デザイン・レイアウト仕様
    - CDプラケース台座（長方形アート対応）
    - 長方形の画像が来ても絶対にトリミング（クロップ）しない。
    - ベースとして「正方形の少し暗い半透明のガラス板（CDのジュエルケース裏面）」を描画し、その中にアスペクト比を維持してアートを収める（レターボックス化）。
    - 余白部分は「ケースの透明なプラスチック部分」としてエモい質感を放つ。
    - CDの「帯（オビ）」スタイルのトラックNo
    - トラックNo（例: 015 / 200）は、プラケースの左側に「帯」のように縦書き（-90度回転）で配置する。下側を先頭（読む方向）とする。
    - 独立した2枚の浮遊プレート（1〜2pxのスリット）
    - プラケース（アート）と帯はピッタリくっつけず、**1〜2pxの極小の隙間（スリット）**を空けて配置する。
    - それぞれが独立してドロップシャドウを落とすことで、「2枚の高級なガラスプレートが絶妙なクリアランスを保って宙に浮いている」ような極上の立体感（アクリル細工感）を演出する。
3. アニメーション仕様（スロット動作）
    - 曲が切り替わった瞬間、「前の曲」の【帯＋プラケース＋アート＋テキスト】のセットが丸ごと上方向へスライドして消えていく。
    - 同時に、「新しい曲」のセットが丸ごと下方向から本来の位置へスライドして現れる。
    - アニメーションの秒数（ねっとり感）は、INIファイルの [Layout_NowPlaying] セクションの TrackInfoScrollDuration でユーザーが自由に調整可能とする。
4. アーキテクチャ・内部実装方針
    - カプセル化の徹底 (Widget_TrackInfo)
    - 曲が切り替わったことを検知したら、現在の表示情報（アート、タイトル、アーティストなど）を「Prevキャッシュ」としてWidget内部に一時退避し、進行度（0.0f〜1.0f）を管理する。
    - 他のファイルや大元の Renderer には一切状態を持たせない（Phase 20で構築した最強のクリーンルームの恩恵）。
    - 見えない巨大コンテナによるクリッピング (LayoutCalculator)
    - LayoutCalculator で、アート・帯・テキストすべてをすっぽり包み込む「見えない巨大な矩形（コンテナ）」を算出。
    - 描画時にそのコンテナで PushAxisAlignedClip（切り抜き）をかけることで、スライド中の要素が絶対に他の領域（アプリアイコンやシークバーなど）にはみ出さないようにする。
