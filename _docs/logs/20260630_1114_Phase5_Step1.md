# 実装レポート: Phase5-1 PlaylistManagerの実装とフルパス再生
作成日時: 2026/06/30

## 目的
再生待ちキューを管理する `PlaylistManager` クラスを新設し、テスト用のMP3をプロジェクトルートの `assets` から「フルパス」で参照して再生する仕組みを構築する。

## 変更・追加ファイル
*   `src/PlaylistManager.h` (新規作成)
*   `src/PlaylistManager.cpp` (新規作成)
*   `src/Application.h` (メンバ追加)
*   `src/Application.cpp` (フルパス解決処理とプレイリスト追加処理の実装)
*   `CMakeLists.txt` (コピー処理の削除、ソースの追加)
*   `PROJECT_ARCHITECTURE.md` (PlaylistManagerの説明追記)

## 実装内容
1.  **PlaylistManagerの作成**:
    ファイルパスのリスト(`std::vector<std::string>`)と現在のインデックスを保持するクラスを作成。`Add` と `GetCurrentTrack` の基本操作を実装した。
2.  **フルパスの解決**:
    `Application::Initialize` 内にて、`GetModuleFileNameW` により実行ファイルパスを取得。そこから `std::filesystem::path` を用いて3つ上のディレクトリ（プロジェクトルート）を求め、`assets/test.mp3` への絶対パスを生成した。
3.  **プレイリストの統合**:
    生成した絶対パスを `PlaylistManager` に登録し、`GetCurrentTrack()` 経由で取得したファイルパスを用いて `TagManager` と `AudioPlayer` を初期化するように修正。
4.  **ビルド設定の更新**:
    `CMakeLists.txt` から実行ファイルディレクトリへの `test.mp3` のコピー処理を削除し、新規作成した `PlaylistManager` のソース・ヘッダをビルド対象に加えた。

## 懸念点・今後の課題
*   現在のフルパス解決ロジックは、実行ファイルが `build/Debug/` や `build/Release/` など、プロジェクトルートから3階層深い場所にあることを前提としている。配布時など実行環境の構成が変わる場合には見直しが必要になる可能性がある。
*   今回は単一のファイルをリストに追加しているだけだが、今後は複数ファイルの追加や、インデックスを移動して次の曲へ進む仕組みなどが必要になる。
