# 実装レポート: Phase 8-5 Hotfix ForceRender

## 目的
Win32のモーダルメッセージループによる描画停止を回避し、ウィンドウのリサイズ中（ドラッグ中）にもリアルタイムで描画を追従させるための強制再描画ロジックを追加する。

## 変更ファイル
* `src/Application.h`, `src/Application.cpp`
  * `Application` クラスに public メソッド `void ForceRender();` を追加。
  * `Run()` メソッド内で毎フレーム実行していた描画直前準備（現在時間の取得、進行度の計算、時間文字列の生成、スペクトルデータの取得）と `Renderer::Render` 呼び出しのロジックを `ForceRender()` 内へ抽出し、`Run()` 内からは `ForceRender()` を呼び出すように変更。
  * `SetOnResizeCallback` のコールバック内において、`m_renderer.Resize` の呼び出し直後に `this->ForceRender()` を呼び出すように修正。
  
* (Window.cpp への直接的な変更はなし)
  * `Window` クラスは `Application` クラスのインスタンスを持たないアーキテクチャ（責務分離）であるため、リサイズイベント(`WM_SIZE`)の処理フローの一環として呼び出されるコールバック(`m_onResize`)内で `ForceRender()` を呼び出すことで、指示要件である「リサイズした直後にForceRenderを呼び出す」を既存のアーキテクチャを崩さずに実現しました。

## 懸念点
* リサイズイベントの最中（OSのモーダルメッセージループ中）に `ForceRender` による描画が発生するため、ドラッグイベントの頻度が高い環境ではメインスレッドの処理負荷が一時的に増大し、音声処理やUIレスポンスに影響を与える可能性があります。
* D2DおよびDXGIスワップチェインの同期（VSync）が働いているため、高速なリサイズ操作時にはディスプレイのリフレッシュレートが律速となり、マウスカーソルの追従遅延を感じる場合があります。
