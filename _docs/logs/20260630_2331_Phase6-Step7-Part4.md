# 実装レポート：常に最背面の維持機構の追加 (Hotfix)

## 1. 実装目的
Phase 6 - Step 7 - part 4 の指示に従い、「Z-Order: 最背面 (Bottom)」設定時に、ユーザーがウィンドウをクリックしたり他のウィンドウを操作した際に手前に出てきてしまう問題（Win32 APIの仕様）を修正し、常に壁紙の直上（最背面）を維持するようにする。

## 2. 変更ファイル
- `src/Window.cpp`

## 3. 変更内容
- `Window::WindowProc` に `WM_WINDOWPOSCHANGING` メッセージの処理を追加。
- `m_config->GetZOrder() == 2`（最背面）の場合に、`lParam` (WINDOWPOS構造体ポインタ) の `hwndInsertAfter` を `HWND_BOTTOM` に強制的に上書きすることで、ウィンドウが手前に浮上するZオーダー変更要求をキャンセルするように実装。
- 同じく最背面設定時において、クリック等による意図しないアクティブ化とそれに伴う挙動不審（タスクバーなどへの影響）を防ぐため、`WM_MOUSEACTIVATE` メッセージを捕捉し、フォーカス奪取を抑制する `MA_NOACTIVATE` を返すように対応。

## 4. 懸念点
- `WM_MOUSEACTIVATE` で `MA_NOACTIVATE` を返すように変更しましたが、ロゴ部分のドラッグ移動（`WM_LBUTTONDOWN` からの `WM_NCLBUTTONDOWN`）は独立して処理されているため、問題なく機能する想定です。
- タスクバーのボタンからのアクティブ化については、`WM_WINDOWPOSCHANGING` が `HWND_BOTTOM` を死守するため、最背面自体は維持されます。
- 現状特に大きな懸念事項はありませんが、運用上もし意図しないフォーカス喪失の問題等が出た場合は、`WM_MOUSEACTIVATE` の制御を緩和する可能性があります。
