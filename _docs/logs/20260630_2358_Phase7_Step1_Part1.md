# 実装レポート：Phase 7 - Step 1 (AudioPlayerの拡張とFFT解析)

## 1. 実装目的
miniaudioから再生中の音声データをキャプチャしてリアルタイムにFFT（高速フーリエ変換）解析を行い、ビジュアライザ（スペクトルアナライザ）描画のための周波数スペクトルデータを抽出する基盤を作成する。

## 2. 変更ファイル
- `src/AudioPlayer.h`
- `src/AudioPlayer.cpp`

## 3. 変更内容
- **音声データのキャプチャ**: miniaudio の `ma_engine_config` における `onProcess` コールバックを活用し、ミックス後の出力PCMデータ（浮動小数点数）をキャプチャする `AudioPlayerOnProcess` 関数を実装。
- **FFT解析の実装**: 外部ライブラリに依存せず、C++ネイティブの Cooley-Tukey Radix-2 FFT アルゴリズムを `AudioPlayer.cpp` の無名名前空間に自作実装。キャプチャしたデータ（ステレオの場合はモノラル化）にハニング窓を適用した上で、1024サンプルごとの周波数スペクトルデータを生成。
- **スレッドセーフなデータ共有**: オーディオスレッドとメインループ（描画スレッド）間で安全にデータをやり取りするため、`std::mutex` を用いて波形データとスペクトルデータへのアクセスを保護。
- **データ取得用メソッド追加**: 最新のスペクトルデータをメインループ側から取得するための `GetSpectrumData(std::vector<float>& outSpectrum)` メソッドを追加。

## 4. 懸念点
- 現状、FFTサイズは1024で固定しており、十分高速に動作しますが、より精細なビジュアライザが必要になった場合はパフォーマンスと精度のバランスを見てサイズ調整や最適化（SIMD化など）を検討する余地があります。
