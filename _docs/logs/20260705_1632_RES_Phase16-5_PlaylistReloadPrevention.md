# RES:実装計画・作業レポート Phase 16-5: 同一プレイリストの再ロード防止 (実装実行)

## 1. 実装目的
プレイリスト一覧モードから「現在再生中のプレイリスト」をダブルクリックして再度開こうとした際、音楽が停止してプレイリストが最初から再読み込みされてしまう挙動を改善する。

## 2. 実装タスクリスト
[x] タスク1: 同一プレイリスト判定と再ロードの防止 (Application)
    - `Application::SwitchPlaylist` において、要求されたファイルパスが現在読み込まれているプレイリストと一致するかどうかを判定するロジックを追加した。
    - パスが一致する場合、内部キューのクリア等の再ロード処理をスキップし、再生を維持したままプレイリスト一覧モードを終了（`m_isPlaylistListViewMode = false;`）するように実装した。

## 3. 詳細作業内容
### タスク1: 同一プレイリスト判定と再ロードの防止 (Application)
- `src/Application.cpp` の `Application::SwitchPlaylist` メソッドの冒頭に、`std::filesystem::equivalent` および文字列の一致による等価性チェックを追加した。
- 等価性が確認された場合は、直ちに `m_isPlaylistListViewMode = false;` を設定し早期リターンすることで、現在の再生やUI状態を保持したまま「曲一覧モード」へ戻る挙動を実現した。
- 合わせて `PROJECT_ARCHITECTURE.md` の `SwitchPlaylist` に関する説明を更新し、同一プレイリストの場合は再ロードが防止される仕様を追記した。
