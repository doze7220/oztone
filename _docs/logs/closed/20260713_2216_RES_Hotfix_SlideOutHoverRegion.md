# RES:実装計画・作業レポート Hotfix: 隠れるUIの格納アニメーション中のホバー領域適正化

## 1. 実装目的
プレイリストやロゴ拡張メニューが格納アニメーション（スライドアウト）している最中に、意図せず広いホバー領域にマウスが触れて再展開してしまう不具合を修正する。

## 3. 実装タスクリスト
- [x] タスク1: 展開状態フラグ（拡張フラグ）の厳格化
    - `src/Widget_Playlist.cpp` と `src/Widget_LogoMenu.cpp` にて展開フラグの算出ロジックを修正。

## 5. HOTFIX1
### 原因・理由: 展開状態フラグがスライドアニメーションの進行度に依存していた
    - プレイリストおよびロゴ拡張メニューが完全に格納されるまで（アニメーション進行中）、「UIが展開している」というフラグが維持されていたため、Window側でのホバー判定領域が広いままになっていた。

### 対応: 展開フラグの条件厳格化
    - `src/Widget_Playlist.cpp` および `src/Widget_LogoMenu.cpp` を修正し、フラグ算出ロジックからアニメーション進行度（`m_playlistSlideX` や `m_menuProgress`）を除外した。
    - これにより、「実際にマウスがホバーしている」または「離脱ディレイタイマーが残っている」状態のみ展開フラグを `true` に設定する（厳格化する）ことで、ディレイ切れ後の格納アニメーション開始と同時に当たり判定領域が本来のサイズへ縮小するようになった。
