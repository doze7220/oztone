# 20260701_2337_Phase8-8_Playlist.md

## 実装目的
Phase 8-8: 「忍者プレイリスト」の実装。シークバー右上の「TRACK XXX/XXX」表示をトリガーとし、ホバー時にウィンドウ右端から現在のシャッフル再生順リストがスライドインするUIを構築する。

## 変更ファイルと詳細な実装内容
### 1. ConfigManager の拡張 (`src/ConfigManager.h`, `src/ConfigManager.cpp`)
- `[Layout_Playlist]` セクションを新設し、以下のパラメータ読み書きとゲッターを追加した。コンストラクタでの初期値設定、`LoadSettings()`、および `SaveDefaultSettings()` でのデフォルト書き出し処理にもすべて反映している。
  - トラック数表示用：`TrackCountFontFamily`, `TrackCountFontSize`, `TrackCountRightOffset`, `TrackCountBottomOffset`, `TrackCountLetterSpacing`, `TrackCountShadowOffsetX`, `TrackCountShadowOffsetY`, `TrackCountShadowOpacity`
  - プレイリストUI用：`PlaylistWidth`, `PlaylistItemOffsetY` (1項目ごとの高さ), `PlaylistTitleFontSize`, `PlaylistArtistTimeFontSize`
  - フォントや配置座標の追加：タイトル用フォント、アーティスト用フォント、背景の不透明度（`PlaylistBgOpacity`）、各種オフセット設定。

### 2. PlaylistManager の拡張 (`src/PlaylistManager.h`, `src/PlaylistManager.cpp`)
- `size_t GetCurrentIndex() const` を実装し、再生中の曲がシャッフルリストの何番目かを取得できるようにした。
- `std::vector<std::wstring> GetShuffleList() const` を実装し、次に再生される順番（シャッフルリスト全体）のファイルパス一覧を取得可能にした。
- 描画時に `playlistManager.GetCurrentIndex()` などを取得し、TRACK表示に反映

## 追加作業タスク1: トラック数表示（TRACK XXX/XXX）の右寄せ対応
- **修正内容**: 
  - `Renderer::Initialize` において、`m_trackCountTextFormat` に対して `SetTextAlignment(DWRITE_TEXT_ALIGNMENT_TRAILING)` を追加で設定しました。
- **理由**:
  - `TrackCountRightOffset` は描画基準の固定枠(幅200.0f)の右マージンとして適用されていましたが、テキストフォーマット自体が右寄せになっていなかったため、枠の中で左寄せになり、結果的に画面右端から意図した以上のマージンが空いてしまっていました。
  - 右寄せ（Trailing）を指定することで、テキストの長さに関わらず、指定されたオフセット分だけ右端から正確に配置されるように修正しました。
  - さらに、トラックカウントのドロップシャドウ描画処理が個別のプロパティ（`TrackCountShadowOffsetX`等）ではなく全体のシャドウ設定を参照していた不具合を修正し、正しくINI設定値が反映されるようにしました。

## 追加作業タスク2: プレイリスト内項目のレイアウト設定分離と再生時間の右寄せ化
- **修正内容**:
  - `ConfigManager`: `PlaylistArtistTime...` 系の設定プロパティを廃止し、アーティスト名用の `PlaylistArtist...` プロパティ群と、再生時間用の `PlaylistTime...` プロパティ群（文字間隔調整 `PlaylistTimeLetterSpacing` 含む）へと分割し、`OZtone.ini` の入出力を対応させました。
  - `Renderer`: プレイリストのテキストフォーマット作成時、アーティスト用 (`m_playlistArtistTextFormat`) と再生時間用 (`m_playlistTimeTextFormat`) に分離しました。再生時間フォーマットには `DWRITE_TEXT_ALIGNMENT_TRAILING`（右寄せ）を設定しました。
  - 描画ループにて、アーティスト名と再生時間の描画処理を独立させました。再生時間はアイテム右下基準のX座標とアイテム左上基準のY座標（`currentY + PlaylistTimeOffsetY`）を起点とし、`IDWriteTextLayout` を用いて文字間隔を適用しつつ右寄せで描画するようにしました。これによりアーティスト名とY座標の並びを合わせやすくなっています。
  - さらに、アーティスト名と再生時間のフォントカラーをINIで個別に指定できるように、`PlaylistArtistColor` と `PlaylistTimeColor` （HEX文字列、例: `#AAAAAA`）の設定プロパティとパース処理を追加し、専用のカラーブラシを適用しました。
- **理由**:
  - 従来はアーティスト名と再生時間が結合されて「Unknown Artist / 00:00」のように単一フォーマットで描画されていたため、個別のフォント変更やレイアウト（特に再生時間のみ右寄せ）が行えないマジックナンバー状態でした。
  - 設定を分離し、再生時間の描画基準を「アイテム右下」とすることで、ユーザーがINIファイルから自由かつ正確に各要素の表示位置やフォントをカスタマイズできるようになりました。

## 追加作業タスク3: 右端ホバー判定幅のINI設定化
- **修正内容**:
  - `ConfigManager` にて `[Layout_Playlist]` セクションに `PlaylistHoverWidth` パラメータを追加し、INIファイルから読み書きできるように対応しました（デフォルト値: 30）。
- **理由**:
  - 右端のプレイリスト表示用のホバー判定幅をハードコードから設定ファイル依存に変更することで、ユーザーが環境に応じてホバーの反応しやすさを調整できるようにするため。

## 追加作業タスク4: Window のホバー判定（IsInPlaylistRegion）の修正
- **修正内容**:
  - `src/Window.cpp` の `IsInPlaylistRegion` の判定ロジックを変更し、X座標が右端から `PlaylistHoverWidth` の範囲内であり、かつY座標が下部の再生コントロール領域（`ControlHoverHeight`）より上の場合のみホバーとして判定するように制限をかけました。
- **理由**:
  - リストが画面下部まで広がる際、右下のリサイズグリップ（15x15）や下部の再生コントロールとのホバー判定領域が干渉し、誤動作を招く恐れがあったため。上部および中間領域の右端のみをプレイリスト表示のトリガーとして明確に分離しました。

## 追加作業タスク5: プレイリストのホバー判定領域の動的拡張
- **修正内容**:
  - `src/Window.cpp` の `IsInPlaylistRegion` メソッドにて、`m_isPlaylistHovered` の状態を評価するように修正しました。
  - プレイリストが開いている（true）場合は、ホバー判定幅を `PlaylistHoverWidth` から `PlaylistWidth`（リスト全体の幅）へ動的に拡張し、リスト領域のどこにマウスがあってもホバー状態を維持できるようにしました。閉まっている（false）場合は従来通り `PlaylistHoverWidth` を使用します。
- **理由**:
  - ホバーでリストが開いた後、マウスをリストの左側に移動させるとホバー判定から外れてしまい、リストが閉じてしまうのを防ぐため。これによりリスト上での操作性が向上します。

## 追加作業タスク6: ホバー優先度の厳格化（排他制御）
- **修正内容**:
  - `src/Window.cpp` における `WM_LBUTTONDOWN` や `WM_MOUSEWHEEL` のイベント処理の冒頭に、`m_isPlaylistHovered` が true の場合は背後にある再生コントロールや音量調整の判定をスキップする（早期リターン）処理を追加しました。
  - `WM_MOUSEMOVE` についても、既存の通りプレイリストホバー時には `m_isControlHovered` 等が強制的に false となり判定がスキップされる仕組みとなっていることを確認・担保しました。
- **理由**:
  - プレイリストが開いている間、背後にあるシークバーやコントロールが反応してしまう不具合（誤操作）を防ぎ、完全なプレイリスト優先（排他制御）状態を実現するため。

## 追加作業タスク7: プレイリスト展開中のホバー判定抜け（Y座標制限）の修正
- **修正内容**:
  - `src/Window.cpp` の `IsInPlaylistRegion` にて、右下UIとの干渉を避けるためのY座標制限を `m_isPlaylistHovered || (logicalY < logicalHeight - controlHeight)` とし、プレイリスト展開中は無効化するように修正しました。
- **理由**:
  - プレイリスト展開中にマウスを画面下部（シークバー付近）へ移動させた際、Y座標制限に引っかかってホバー判定が外れ、意図せずプレイリストが閉じてシークバーが反応してしまう不具合を解消するため。

### 3. Window のホバー判定（優先度制御） (`src/Window.h`, `src/Window.cpp`)
- トラック数表示（TRACK XXX/XXX）領域の矩形をDPIスケーリングを考慮して計算し、`IsInPlaylistRegion(x, y)` メソッドとしてホバー判定を追加し、状態を保持するフラグを追加した。
- `WindowProc` 内のホバー処理において、既存の再生コントロール領域 (`IsInPlaybackControlRegion`) よりも先に `IsInPlaylistRegion` の判定を実行させ、プレイリストポップアップを優先して開くように制御した。

### 4. Renderer の描画とスライドイン・アニメーション (`src/Renderer.h`, `src/Renderer.cpp`)
- トラック数、プレイリストのタイトル、アーティスト＆再生時間表示用にそれぞれの `IDWriteTextFormat` を初期化処理 (`Initialize`) で作成。
- `Render` の最後尾にて、「TRACK XXX/XXX」のテキストを影、トリミング、文字間隔を適用して描画する処理を追加。
- 毎フレームの描画処理において、`isPlaylistHovered` をもとにスライドイン状態 (`m_playlistSlideX`) をイージングアニメーションで更新し、スライド板を描画。
- プレイリストの背景（半透明黒板）を描画後、`PushAxisAlignedClip` でクリッピングを設定し、アイテムが領域外へはみ出さないようにした。
- 現在の再生曲がリスト中央に来るようスクロールY座標を計算し、末尾と先頭で余白ができすぎないよう `std::clamp` を用いてスクロール範囲を制限。
- 各項目は `DirectWrite` で描画し、現在の再生曲（`i == currentTrackIndex`）の背景には白の半透明ブラシを描画してハイライト表示し、`std::filesystem::path` を用いてファイルパスからファイル名（タイトル）を抽出して描画した。

### 5. Application の連携 (`src/Application.cpp`)
- `Application::ForceRender` と `Application::Run` において、`PlaylistManager` から「現在のインデックス」「総曲数」「シャッフルリスト順」を取得し、それを `Renderer::Render` に毎フレーム渡すように引数と橋渡し処理を変更した。

## 懸念点・備考
- リスト描画時のスクロール制御において、曲数が多い場合に備え `minScroll` をアイテム高さを基に動的に計算し、はみ出しを防止している。
- 開発途中で `Renderer.cpp` に対するファイル更新時のマージミスがあり、EndDraw 等のブロックが重複する問題が発生したが、該当箇所の削除と整理を行い、不足していた `<filesystem>` のインクルードと ConfigManager の設定読み込みを追記することで、正常にビルドを通過させた。
