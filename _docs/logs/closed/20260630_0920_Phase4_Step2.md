# 実装レポート: Phase4 Step2 (AudioPlayer Update)

## 実装目的
AudioPlayerクラスの再生方式を従来の fire-and-forget 方式（`ma_engine_play_sound`）から、`ma_sound` オブジェクトを用いた管理方式へ移行しました。これにより、再生中の状態管理および再生時間の取得（現在位置・総時間）が可能になりました。

## 変更ファイル
- `src/AudioPlayer.h`
  - `miniaudio.h` のインクルード追加
  - `ma_sound m_sound`、`bool m_isSoundLoaded` メンバ変数の追加
  - 再生時間・状態取得用のメソッドプロトタイプ（`GetPositionSeconds`, `GetLengthSeconds`, `IsPlaying`）の追加
- `src/AudioPlayer.cpp`
  - コンストラクタでの初期化追加
  - `Uninitialize` メソッドにて `ma_sound_uninit` を呼び出すように変更（リソースリークの防止）
  - `Play` メソッドを `ma_sound_init_from_file` および `ma_sound_start` による実装へ改修
  - 各種取得メソッド（`GetPositionSeconds`, `GetLengthSeconds`, `IsPlaying`）の実装
- `PROJECT_ARCHITECTURE.md`
  - `AudioPlayer` クラスのリファレンスセクションに、新しく追加された関数の仕様と動作変更内容を追記

## 懸念点・特記事項
- 現在の `ma_sound_get_cursor_in_seconds` および `ma_sound_get_length_in_seconds` は正常に動作しますが、一部のフォーマット（VBRのMP3など）では総時間が正確に取得できない場合があります。もし精度問題が発生した場合は別途タグ解析（TagLibなど）を用いて曲長を取得するアプローチの検討が必要です。
- 再生終了（EOF）の検知ロジックは今回未実装です（`IsPlaying()` が false になるかどうかで判定は可能ですが、コールバック等は設定していません）。必要になったタイミングで `ma_sound_set_end_callback` などの導入を検討してください。
- `build.bat` による自動ビルドは正常にパスしました。
