# 実装レポート: Phase 8-6 ウィンドウ最小サイズ制限の実装 (2026-07-01 20:57)

## 実装目的
ウィンドウの限界サイズ（最小サイズ）をハードコードし、OSレベルでそれ以上縮まらないように制限をかける。

## 変更ファイル
*   `src/Window.cpp`
    *   `WindowProc` 内に `WM_GETMINMAXINFO` メッセージの処理を追加。
    *   `lParam` を `MINMAXINFO*` にキャストし、現在のウィンドウDPIを取得（取得できない場合は96をデフォルトとする安全策を適用）。
    *   論理限界サイズ (Width: 495, Height: 260) を現在のDPIに基づいて物理ピクセルサイズに変換し、`ptMinTrackSize.x` および `ptMinTrackSize.y` に設定した。

## 懸念点
*   特になし。DPI変更時の再計算も、`WM_GETMINMAXINFO` がリサイズ時に都度呼び出されるため、常に現在のDPIに基づいて正しくスケーリングされた最小サイズが適用されます。
