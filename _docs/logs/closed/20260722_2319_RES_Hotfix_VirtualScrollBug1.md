# RES:HOTFIX作業レポート: 仮想スクロールバグ修正1 (ホバー領域・ピン留め判定の適正化)

## 1. 実装目的
仮想スクロール実装に伴い発生した、TrackInfo領域のホバー判定が過剰に広すぎるバグ（バグ1）と、プレイリスト展開時の領域拡張ロジックがピン留め時にも発動して他のUIへのホバーを阻害するバグ（バグ4）を修正する。

## 2. 調査内容
1. `Window_Mouse.cpp` の `IsInTrackInfoRegion` のY座標上限判定において、`- 40.0f` のハードコードされた余白が含まれており、これが判定領域を無駄に上部へ拡張していた。
2. `Window_Mouse.cpp` の `IsInPlaylistRegion` のY座標制限解除ロジック（`isYMatch`）において、プレイリスト展開中かどうかの判定のみが行われており、ピン留め中であることが考慮されていなかった。そのため、ピン留め中かつ展開状態の場合に画面下部への過剰なY判定拡張が行われていた。

## 3. 対象ファイル
* `src/Window_Mouse.cpp`

## 4. 実装タスクリスト
[x] タスク1: TrackInfoホバー領域の適正化 (バグ1対応)
[x] タスク2: プレイリストピン留め時の領域専有の解消 (バグ4対応)

## 5. 詳細作業内容
* タスク1: TrackInfoホバー領域の適正化
    - `IsInTrackInfoRegion` 内の `topLimit` 計算から `- 40.0f` を削除し、アルバムアートとテキストが描画される実際の高さ（`logicalHeight - BaseBottomOffset - ArtSize`）に厳格に制限しました。
* タスク2: プレイリストピン留め時の領域専有の解消
    - `IsInPlaylistRegion` 内の `isYMatch` 判定条件に `!m_config->GetIsPlaylistPinned()` を追加し、ピン留め時は展開状態であっても過剰なY座標拡張を行わず、Y座標制限が適用されるように修正しました。
