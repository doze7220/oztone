# 実装レポート: シャッフルボタンの動的アイコン切り替えとトグル演出の廃止

## 1. 概要
ロゴ拡張メニューのシャッフルボタンについて、トグルOFF時の「赤い斜線」での表現を廃止し、シャッフルモードに応じて動的にアイコンとテキストが切り替わる仕様へ変更しました。

## 2. 修正の原因（目的）
シャッフルOFF時は「ストレート（順次）再生」という対等なモードであり、機能が無効化されているわけではないため、斜線でのトグルOFF表現ではなく、アイコンそのもの（「🔀」と「➡️」）を切り替える表現が適切であると判断されたためです。

## 3. 詳細作業内容

### `src/Window.cpp` の修正
* コンストラクタ内の `m_logoMenuItems` リストにおいて、シャッフルボタン（`ID_LOGO_SHUFFLE`）の `isToggle` 設定を `true` から `false` に変更しました。これにより、ボタンクリック時に内部のトグル状態による斜線描画が発動しないようになりました。

### `src/Widgets.cpp` の修正
* `LogoMenuWidget::Draw` メソッド内において、メニューアイテムを描画するループに介入し、アイコンテキストを動的に上書きする処理を追加しました。
  * `commandId` が `ID_LOGO_SHUFFLE` の場合、`config->GetShuffleMode()` を評価し、`true` なら `L"🔀"`、`false` なら `L"➡️"` を描画テキスト（`iconText`）として設定するようにしました。
* 同様に、ロゴ拡張メニュー上部の機能説明テキストについても動的切り替えを追加しました。
  * ホバー中のアイテムがシャッフルボタンの場合、`config->GetShuffleMode()` を評価し、`true` なら `L"再生モード：シャッフル"`、`false` なら `L"再生モード：プレイリスト登録順"` を表示するように修正しました。

### `PROJECT_ARCHITECTURE.md` の修正
* ロゴ拡張メニューの仕様説明部分を更新し、シャッフル切替ボタンが「🔀」と「➡️」で動的に切り替わる旨を追記しました。また、トグルOFF時の赤い斜線表現についての説明に、動的切替を行う例外事項を付記しました。
