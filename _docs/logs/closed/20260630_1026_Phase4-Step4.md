# Phase 4-4: TagLib Integration and UI Improvements (Implementation Log)

## 実施内容
1. **TagLib v2.0.2 の導入 (CMake)**
   - `FetchContent` を利用して TagLib v2.0.2 を静的ライブラリ (`TAGLIB_STATIC`) として組み込みました。
   - インクルードパス (`_deps/taglib-src` 以下) や DLLインポート宣言の回避を行い、ビルドが通るように修正しました。

2. **TagManagerクラスの実装**
   - `src/TagManager.h`, `src/TagManager.cpp` を作成。
   - `TagLib::MPEG::File` および `TagLib::ID3v2::Tag` を利用してMP3からタイトル・アーティスト名の抽出を実装しました。
   - `TagLib::ID3v2::AttachedPictureFrame` からバイナリデータ (APIC) を抽出し、アルバムアートとしてメモリ上に保持する処理を実装しました。

3. **Rendererクラスの拡張**
   - 文字列 (曲名・アーティスト名) を動的に描画するため、`m_currentTitle`, `m_currentArtist` 等のメンバ変数を追加しました。
   - メモリからの画像読み込み (`LoadBitmapFromMemory`) メソッドを追加し、WIC のストリーム作成 (`CreateStreamFromMemory`) を介して動的に `ID2D1Bitmap` を生成するようにしました。
   - `SetTrackInfo`, `SetAlbumArt` メソッドを追加し、外部からこれらの情報をセットできるようにしました。
   - 描画時、ダミーテキストではなく変数内の値を用いて `IDWriteTextLayout` を再生成するよう処理を更新しました。

4. **Applicationクラスとの連携**
   - 初期化時の再生ルーチンにおいて、`TagManager` インスタンスをローカル生成して `assets/test.mp3` を読み込み、得られた情報を `Renderer` の `SetTrackInfo` / `SetAlbumArt` に渡す実装を行いました。

## 発生した課題と解決策
- **コンパイルエラー（前方宣言や依存ヘッダの不足）**:
  - `Renderer.h` 内で `std::vector` を使用しているにもかかわらず `<vector>` のインクルードが欠落していたため、多数の構文解析エラーが発生しました。これを修正しました。
- **TagLib ヘッダの include パス問題**:
  - FetchContent で取得した TagLib 2.0.2 は `target_include_directories` で内部構造を自動公開しない構成であったため、`_deps/taglib-src` 以下の各サブディレクトリ (mpeg, mpeg/id3v2, toolkit など) を手動でインクルードパスへ追加して解決しました。
- **リンクエラー（dllimport）**:
  - WindowsでTagLibを静的ビルドする際、ヘッダに `TAGLIB_STATIC` を定義しないとデフォルトで DLL のインポートと見なされ、多数の未解決シンボルエラーが発生します。`CMakeLists.txt` に `target_compile_definitions(OZtone PRIVATE TAGLIB_STATIC)` を追加することで解決しました。

## 確認状況
- エラーはすべて解消され、`build.bat` によるビルドが正常に終了することを確認しました。
- `PROJECT_ARCHITECTURE.md` に `TagManager` の詳細を追記しました。
