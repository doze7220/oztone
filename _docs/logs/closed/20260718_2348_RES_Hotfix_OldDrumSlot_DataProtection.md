# RES:HOTFIX作業レポート: プレイリスト切り替え時のOLDメタ情報入れ替わり修正

## 1. 実装目的
プレイリスト切り替え中に曲のタイトルやアーティスト名などのテキストメタ情報が正しく抽出されず、過去の器（OLDスロット）の情報が上書きされてしまう不具合を修正し、ドラムスロット（カレント・OLD）から画像とともにテキスト情報も直接抽出する堅牢なロジックを実装すること。

## 2. 調査内容
描画ループ内において、旧実装では画像のみを `DrumSlot` から抽出し、テキスト情報については常に `shuffleMetadataList` から動的に再計算して取得していた。プレイリストが切り替わった場合、相対位置からの計算では内容が異なってしまうため、特にアニメーション中のOLDスロットで描画されるべき過去のテキスト情報が失われていた。
そのため、カレントおよびOLDスロットに関しては `DrumSlot` 内部に保存されている `trackTitle`, `trackArtist`, `trackNumber` を直接読み取るように抽出ロジックを分岐させることで対応する方針とした。

## 3. 対象ファイル
* `src/Widget_TrackInfo.cpp`

## 4. 実装タスクリスト
[x] タスク1:視界データ抽出ロジックの堅牢化 (DrumSlotデータの絶対保護) - カレントおよびOLDスロットのテキスト抽出を DrumSlot 直接アクセスに変更。
[x] タスク2:描画の単一フロー維持 - 抽出後の描画は単一フローで実行されるよう維持。

## 5. 詳細作業内容
* タスク1:視界データ抽出ロジックの堅牢化 (DrumSlotデータの絶対保護)
    - `Widget_TrackInfo.cpp` におけるテキストメタ情報の抽出処理について、`relativeIndex == 0` （カレント）および `m_animatingOldIndexOffset` （OLDスロット）に合致する場合、動的計算を行わずに `DrumSlot` (それぞれ `ctx.currentDrumSlotIndex`, `1 - ctx.currentDrumSlotIndex`) に保持されている情報を優先的に使用するように修正した。
    - 中間スロット（ドラムアニメーション中の補間用ダミー等）については、画像の抽出を行わずテキストのみ `shuffleMetadataList` と相対インデックスから動的計算するロジックとして `else` ブロックに分離した。
* タスク2:描画の単一フロー維持
    - データ抽出後の描画処理については `if` 分岐を追加せず、単一の描画フローを維持するよう構成した。
