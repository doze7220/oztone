# 実装レポート: Phase8-3_AddFileType

## 実装目的
- ドラッグ＆ドロップで受け付ける対応ファイルフォーマットの拡張。
- `miniaudio` が公式対応している `.flac`, `.wav`, `.ogg` をプレイリスト追加の許可対象とし、さらに実験的対応として OS の Media Foundation フォールバックによる再生を期待する `.mp4`, `.m4a` を許可対象とする。
- 許可拡張子ごとにマジックナンバー（ファイルヘッダ）を検証し、正しいフォーマットかどうか判定するロジックを実装。ただし、実験的枠の動画コンテナ形式 (`.mp4`, `.m4a`) は検証が複雑であるため、マジックナンバーチェックをスキップさせる。
- 実験的対応部分は、将来的に容易に削除・改変できるように `// [EXPERIMENTAL] MP4/M4A Support` コメントで隔離する。

## 変更ファイル
- `src/Application.cpp`
  - `IsMp3File` を `IsSupportedAudioFile` にリファクタリングし、許可拡張子を追加。
  - `IsValidMp3` を `IsValidAudioFile` にリファクタリングし、`FLAC`, `OGG`, `WAV` のマジックナンバー検証を追加。
  - `OnFilesDropped` 内部で呼び出されるチェック関数の名前を変更し、ログ出力を修正。

## 懸念点
- `.mp4` や `.m4a` はマジックナンバーのチェックを完全にバイパスしているため、単に拡張子を偽装しただけのファイルでもプレイリストに追加されてしまう可能性があります。
- miniaudio の OS フォールバック機能（Media Foundation等）が環境によっては動作せず、再生に失敗してスキップされる場合があります。
- 動作確認（FLAC, WAV, OGGの再生・タグ読み込み、MP4の音声再生）はコンパイルの成功をもって実装完了としていますが、実際の音声出力やタグの表示はローカル環境での手動テストを推奨します。
