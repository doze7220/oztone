# RES:実装計画・作業レポート Phase Hotfix: ロゴ拡張メニュー展開アニメーションの完走

## 1. 実装目的
ロゴ拡張メニュー（LogoMenuWidget）がスライド展開している最中にマウスホバーが外れた際、アニメーションが途中で不自然に停止してしまう不具合を修正する。

## 2. アーキテクチャ設計
### 要件1: アニメーション進行条件の修正
    - ロゴ拡張メニューの維持ディレイ中（m_logoMenuLeaveTimer > 0.0f）は、マウスホバーが外れていてもメニューの展開アニメーション（m_menuProgressの加算）を継続し、最後まで完走させるようにする。

## 3. 実装タスクリスト
[x] タスク1: アニメーション進行条件の修正
    - `src/Widget_LogoMenu.cpp` の `UpdateAnimation` を修正し、維持ディレイ中でも展開アニメーションが進行するよう変更。

## 4. 詳細作業内容
### タスク1: アニメーション進行条件の修正
    - `src/Widget_LogoMenu.cpp` 内の `UpdateAnimation` メソッドにおける `m_menuProgress` の更新ロジックを分離・変更。
    - ホバー状態またはディレイ時間中であれば加算を行い、それ以外の場合に減算を行うよう条件分岐を再構築した。イージングや速度計算は既存のまま維持した。

## 5. HOTFIX1
### 原因・理由: メニュー展開アニメーションの中途停止
    - `UpdateAnimation` メソッド内で、メニュー展開（`m_menuProgress`の加算）の条件がマウスホバー状態（`ctx.isLogoMenuHovered || ctx.isHovered`）のみに依存していたため、展開途中でマウスが外れると加算処理がスキップされ、かつディレイ中は減算も行われないため、進行度が途中でストップする不具合が発生していた。

### 対応: アニメーション進行条件の変更
    - タイマーの減少ロジックとアニメーション進行のロジックを分離し、`m_logoMenuLeaveTimer > 0.0f` である維持ディレイ期間中も `m_menuProgress` を加算（展開）するように修正した。これによりマウスが外れてもスムーズに展開アニメーションが完走するようになった。
