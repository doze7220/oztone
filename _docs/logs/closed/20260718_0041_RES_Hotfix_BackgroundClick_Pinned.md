# RES:HOTFIX作業レポート: ピン留め時の背景クリック判定抜け修正

## 1. 実装目的
プレイリストのピン留め時（排他制御の緩和時）に、純粋な背景クリックによる能動的フェードアウト（ディレイキャンセル）が発動しなくなっている不具合を修正する。

## 2. 調査内容
ピン留め中は `m_isPlaylistHovered` が常に `true` となる仕様のため、従来の `!m_isPlaylistHovered` を含んだ排他制御ブロックやヒットテストの先頭チェックにおいて、意図せずブロック全体がスキップされる状態になっていた。
このため、他のUI領域判定にも引っかからなかった場合のフォールバックとして設定されていた背景クリックの判定が漏れてしまっていた。

## 3. 対象ファイル
* `src/Window_Mouse.cpp`

## 4. 実装タスクリスト
- [x] タスク1: HandleLButtonDown へのフォールバック発火追加
- [x] タスク2: HandleRButtonDown へのフォールバック発火追加

## 5. 詳細作業内容
* タスク1: HandleLButtonDown へのフォールバック発火追加
    - 既存のヒットテストの最後尾に `else if` ブロックを追加し、いずれのUI領域にも該当しなかった場合（`!IsInPlaylistRegion` かつ他の領域外）に、`m_onBackgroundClickCallback()` を呼び出すように修正した。
* タスク2: HandleRButtonDown へのフォールバック発火追加
    - 背景の右ドラッグ処理（フレーミング操作）を開始する直前の判定条件について、`!m_isPlaylistHovered` を `!IsInPlaylistRegion(xPos, yPos)` に置き換え、ピン留め中であっても背景領域であれば確実にフォールバック処理（背景クリックコールバック呼び出し）が実行されるように修正した。
