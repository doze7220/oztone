# RES:HOTFIX作業レポート: スクロール中のメタ情報消失とOLDハードコード修正

## 1. 実装目的
TrackInfoWidgetにおける、中間スロット（ドラムアニメーション中のスクロール）でのメタ情報の消失と、インデックスに基づく描画フローのハードコード（`i == 0`, `i == -1`など）を解消し、データの抽出と描画実行を完全に分離した単一の共通フローへと純化すること。

## 2. 調査内容
以前の実装では、描画ループ内でカレント (`relativeIndex == 0`) 用のキャッシュを利用する処理と、OLD用 (`relativeIndex == m_animatingOldIndexOffset`) に動的レイアウト生成する処理が、`if`文によって描画のフローごと分離されていました。その結果、中間スロット（両者以外）ではテキストが適切に生成・描画されていませんでした。
これを解決するため、描画の直前で全スロットに対して無条件に「視界データ（テキストと画像）」を抽出・レイアウト生成（キャッシュが一致すればキャッシュ利用）するロジックに一本化し、描画ロジックから `i` に依存した分岐を完全撤廃する必要がありました。

## 3. 対象ファイル
* `src/Widget_TrackInfo.cpp`

## 4. 実装タスクリスト
[x] タスク1:描画フローからの i=x 条件分岐の完全撤廃 (カメラの純化) - 描画ループ内の `if` 分岐を削除し、データ抽出と描画を分離。
[x] タスク2:視界データの抽出 (データ準備と描画の分離) - 常に `ctx.shuffleMetadataList` と相対位置からテキストを取得し、キャッシュ一致判定によってレイアウトを利用または動的生成。
[x] タスク3:単一フローによる描画の実行 - 抽出した画像およびテキストデータを単一フロー内で無条件に描画。

## 5. 詳細作業内容
* タスク1:描画フローからの i=x 条件分岐の完全撤廃 (カメラの純化)
    - `TrackInfoWidget::Draw` 内の `drawDrumItem` ラムダ式において、描画実行部分が `relativeIndex == 0` かどうかで分かれていた構造を解消しました。
* タスク2:視界データの抽出 (データ準備と描画の分離)
    - 描画ループの先頭で `ctx.shuffleMetadataList` を用いて、`relativeIndex` に対応する正規化インデックスからメタデータ文字列を抽出するようにしました。
    - 抽出した文字列が直前のキャッシュ（`m_lastTitle` など）と一致する場合は `m_titleTextLayout` などのキャッシュを流用し、異なる場合は動的に `IDWriteTextLayout` を生成する共通ロジックに統合しました。
    - 画像データについては、カレントまたはアニメーション中のOLDオフセットに合致する場合のみ `ctx.drumSlots` から取得し、それ以外は `nullptr` とするようにしました。
* タスク3:単一フローによる描画の実行
    - 抽出完了後、画像が `nullptr` の場合はフォールバック（ガラス板）を描画し、取得されたテキストレイアウトも単一のフローで描画されるようにしました。これにより、スクロール中の中間スロットでもテキストが表示されるようになります。

### HOTFIX1
#### 原因・理由: TrackMetadata の構造とポインタアクセスの誤りによるビルドエラー
    - `ctx.shuffleMetadataList` がポインタであるにも関わらずドット演算子でアクセスしていたこと、および `TrackMetadata` 構造体のメンバー名を誤って指定（`trackTitle` -> `title`, `trackArtist` -> `artist`）していたため。
    - また、`TrackMetadata` には `trackNumber` が存在しないため、トラック番号の文字列生成処理を再実装する必要がありました。

#### 対象ファイル: src/Widget_TrackInfo.cpp

#### 対応: ポインタのデリファレンスと構造体メンバー名の修正、フォーマット処理の追加
    - `ctx.shuffleMetadataList` をデリファレンスしてアクセスするように修正しました。
    - メタデータのメンバー参照を正しく `title`, `artist` に修正しました。
    - `trackNumber` は `ctx.shuffleIndices` と `ctx.totalTracks` を利用して `swprintf_s` を用いて "001/050" の形式で動的に生成する処理を追加しました。
    - また、一部変数の二重定義エラー（変数の再宣言）も併せて解消しました。
