# 実装レポート：Phase 1 - Step 2 (ConfigManager導入とウィンドウスタイル動的変更)

## 実装目的
設定ファイル (`OZtone.ini`) を管理する `ConfigManager` クラスを新規導入し、それを用いてアプリケーション起動時のウィンドウスタイル（タイトルバーの有無、境界線・リサイズ枠の有無、タスクバー表示の有無）を動的に切り替えられるようにする。

## 実装詳細
*   `ConfigManager` クラスを作成し、Win32 API (`GetPrivateProfileIntW`, `WritePrivateProfileStringW`, `GetModuleFileNameW`) を用いて実行ファイルと同階層にある `OZtone.ini` の読み込みおよび新規作成（デフォルト値0）を実装。
*   `Application` クラスのメンバに `ConfigManager` を追加し、`Initialize` にて初期化を実行。
*   `Window::Initialize` メソッドに `ConfigManager` を参照として渡し、その設定値に基づいて `dwStyle` (WS_OVERLAPPEDWINDOW, WS_POPUP 等) と `dwExStyle` (WS_EX_TOOLWINDOW, WS_EX_APPWINDOW) を動的に切り替えるよう実装。

## 懸念点
*   現在のターミナル環境（PowerShell）にはビルドツールチェイン (CMake, MSVC等) へのパスが通っていないため、エージェント環境での自動ビルドテストは実施できていません。ユーザ側でのビルドをお願いします。
*   現状、完全な枠なし (`WS_POPUP`) ウィンドウになった場合、ウィンドウのドラッグ移動や終了処理（×ボタンがないため）のための手段がありません。これは後続のフェーズで実装されることを想定しています。
