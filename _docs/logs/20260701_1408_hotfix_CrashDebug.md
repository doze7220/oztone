# 実装レポート：起動時クラッシュの修正 (Hotfix 9)

## 実装目的
Phase 7で実装したビジュアライザにおいて、起動直後（No Track状態）に空のスペクトルデータへアクセスしてしまい、Debugビルドでアサーションエラー（強制終了）が発生する問題を修正する。

## 変更ファイル
- `src/Visualizer.cpp`

## 変更内容
- `Visualizer::DrawPrismBeat` および `Visualizer::DrawCircleParticle` メソッドの先頭に、スペクトルデータが空あるいは要素数が少ない場合（`spectrum.size() < 2`）に即座に `return` するガード句を追加しました。
- これにより、曲が読み込まれていない起動直後の「No Track」状態において、配列外アクセスやアサーションエラーによるクラッシュが発生しないようにしました。

## 懸念点
- 現状は `spectrum.size() < 2` の場合に描画をスキップしていますが、ビジュアライザの実装仕様上、これによって不都合が発生することはありません（最低でも2つの要素がないと補間等が正しく行えないため）。
- 正常に起動し、待機状態を維持できることを確認しました。
