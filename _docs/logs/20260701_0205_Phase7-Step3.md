# 実装レポート: CircleSpectralAnalyzerの実装 (Phase 7-3)

## 実装目的
現在の直線型オシロスコープ (PrismBeat) に加え、新しい「円形パーティクル・スペアナ (Halo Dust)」を実装し、システムトレイメニューから動的に切り替えられるようにする。また、曲ごとに異なる固有のテーマカラーを持たせ、振幅に応じて3D回転するパーティクルを放出するビジュアル表現を構築する。

## 変更ファイル
- `src/ConfigManager.h`, `src/ConfigManager.cpp`:
  - `m_visualizerMode` を追加し、`[Visualizer]` セクションから INI ファイルへ読み書きを行う機能を追加。
- `src/resource.h`, `src/Window.cpp`:
  - システムトレイメニュー用のリソースID `ID_TRAY_VIS_PRISM`, `ID_TRAY_VIS_CIRCLE` を追加。
  - メニューアイテムの構築処理と `WM_COMMAND` でのモード切り替え・保存処理を追加。
- `src/Visualizer.h`, `src/Visualizer.cpp`:
  - 描画メソッドを `DrawPrismBeat` と `DrawCircleParticle` に分離。
  - `DrawCircleParticle` の実装において、曲名とアーティスト名のハッシュ値を用いた固有カラーの生成ロジックを追加。さらにベースカラー(#888888)との加算合成（クランプ）処理を入れることで、白く眩しく発光する「真のネオン質感」を表現。
  - 振幅の減衰を検知してパーティクル（`Particle` 構造体）やレーザー(`LaserRay`)を放出するシステムを追加。
  - 放出されたパーティクルに対し、ロドリゲスの回転公式を用いた3D軸回転計算を行い、2Dポリゴンとして投影・描画する処理を実装。
  - バーとパーティクルの枠線に対して、ネオングロー色と1pxの実線（コア）を重ねる2パス描画を実装し、ネオンのような質感を付与。
  - パーティクルおよびレーザーの射出角度を周波数帯の角度に依存させず、360度ランダムな方向に飛散するように修正（低音域の右上に偏る問題を解消）。
  - `Visualizer.cpp` 冒頭に「ビジュアル設定」セクションを設け、波形振幅、減衰・強調係数、速度、寿命、透明度、ネオンの太さなどを `constexpr` で抽出し、調整を容易にした。
- `src/Renderer.cpp`:
  - `Renderer::Initialize` にて `Visualizer` に `ConfigManager` を渡す処理を追加。
  - `Renderer::Render` から `Visualizer::Draw` を呼び出す際に、曲情報（`trackTitle`, `trackArtist`）を引数として渡すよう変更。

## 懸念点・特記事項
- 円形ビジュアライザを描画する際、D2Dの `SetTransform` を用いてキャンバスを回転させていますが、Renderer側ですでにDPIスケーリングが適用されているため、描画前後に `GetTransform` と `SetTransform` を用いて元の行列を保存・復元する手順を採用しています。これによりDPIの不具合（拡大率の喪失）を防止しています。
- ロドリゲスの回転公式による投影描画は、CPU上で毎フレーム行列計算を行って `ID2D1PathGeometry` を再構築しているため、パーティクル数が極端に増えた場合のパフォーマンス低下に注意が必要です。現状はパーティクルの寿命と発生頻度を調整して負荷を抑えています。
