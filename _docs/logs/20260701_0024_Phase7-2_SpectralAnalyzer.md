# 実装レポート: Phase 7 - Step 2 (ビジュアライザ分離と7色ネオン心電図・高解像度版)

## 実装目的
スペクトルデータを元に、「7色ネオンの心電図（オシロスコープ）風ビジュアライザ」を実装する。また、低音域の解像度不足を解消するためにFFTサイズを大幅に引き上げ、対数マッピングと線形補間を導入する。

## 変更ファイル
- `src/AudioPlayer.cpp`
  - FFTサイズ(`FFT_SIZE`)を1024から8192に引き上げ。
- `src/Renderer.h`
  - `Visualizer` クラスのインスタンス (`m_visualizer`) を追加。
- `src/Renderer.cpp`
  - 仮実装のスペアナ描画コードを削除。
  - 背景アートの描画とUI（アイコン・曲情報等）の描画の間に、`m_visualizer.Draw` の呼び出し処理を追加（適切なZ順を担保）。
- `src/Visualizer.h` (新規作成)
  - 描画を担当するビジュアライザクラスの定義。
- `src/Visualizer.cpp` (新規作成)
  - 横幅を1200分割し、両端5%をマージン領域として設定。
  - マージン領域では再生中に限り微小なランダムノイズ（アナログノイズ）を加算。
  - 中央90%領域では、スペクトルデータを人間の耳に合わせた対数スケールにマッピングし、小数点インデックスから線形補間（Lerp）を用いて滑らかな値を取得。
  - 取得した振幅を用いて、隣り合う頂点のY座標を交互に反転（上・下）させて心電図の脈打ちを表現。
  - 描画領域を7等分し、「赤、オレンジ、黄、緑、水色、青、紫」の7色でパキッと色を割り当て。
  - ネオン効果として、太い半透明ブラシ（グロー）と細い不透明ブラシ（芯）の2回描画を実装。
- `CMakeLists.txt`
  - 新規作成した `Visualizer.cpp` と `Visualizer.h` をビルド対象に追加。

## 懸念点・特記事項
- FFTサイズを8192に引き上げたことにより、FFT計算の負荷（CPU使用率）がわずかに増加しています。現在はC++ネイティブ実装のFFT（`PerformFFT`）を使用しているため、今後のフェーズでパフォーマンスに影響が出る場合は、最適化を検討する余地があります。
- `std::pow` などの数学関数のループ内呼び出しが含まれています。現時点の頂点数（1200）では問題になりませんが、より高フレームレート・高解像度を狙う場合は、対数マッピングの計算結果を事前キャッシュ（テーブル化）することで高速化が可能です。
- ビルド時に `AudioPlayer.cpp` で警告（暗黙の型変換やデータ消失の可能性など）がいくつか出ていますが、元の1024ベースの実装時と同一ロジックであり、動作に実害はないため今回は型変換警告の抑制や修正は見送っています。必要に応じて警告解消のリファクタリングを行えます。
