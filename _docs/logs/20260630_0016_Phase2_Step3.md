# 開発ログ: 2026-06-30 00:16 Phase 2 - Step 3

## 実装目的
Phase 2 - Step 3 として、WICを用いたプレースホルダー用アルバムアート画像の読み込みと、背景全面へのCover表示および左下への配置描画を実装しました。また、`ConfigManager`を通してiniファイルからレイアウト情報や不透明度を取得するデータ駆動のUI設計を導入しました。

## 変更ファイル
- `src/resource.h` / `src/OZtone.rc`
  - プレースホルダー画像(`placeholder_art.png`)をリソースID `IDI_PLACEHOLDER_ART` として登録。
- `CMakeLists.txt`
  - ビルド後のカスタムコマンドに `placeholder_art.png` を出力ディレクトリへコピーする処理を追加。
- `src/ConfigManager.h` / `src/ConfigManager.cpp`
  - `[Layout_NowPlaying]` セクションの項目 (`BaseX`, `BaseY`, `ArtOffsetX`, `ArtOffsetY`, `ArtSize`, `BgOpacity`) の読み書きとゲッターを追加。
- `src/Renderer.h` / `src/Renderer.cpp`
  - `m_placeholderArtBitmap` のロード処理を追加。
  - `Render()` メソッド内で、背景のCover表示（センタリング・トリミング）と左下の指定座標への描画を `D2D1_BITMAP_INTERPOLATION_MODE_LINEAR` を用いて実装。
- `PROJECT_ARCHITECTURE.md`
  - 実装状況を Phase 2 Step 3 へと更新。

## 懸念点・今後の課題
- 実際のMP3ファイルからTagLib等を用いてアルバムアートを抽出する実装は今後のフェーズで対応予定です。
- 背景描画時、ウィンドウサイズが変更された場合でもCover計算が動的に行われるように `Render` の毎フレームの描画処理内でレンダーターゲットのサイズを取得して計算を行っていますが、サイズ変更がない場合は事前に計算結果をキャッシュすることでパフォーマンス向上が見込めます。
