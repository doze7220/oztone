# RES: 実装計画・作業レポート Phase 21-4: アーキテクチャ資料の極限スリム化と目的の明記

## 1. 実装目的
`PROJECT_ARCHITECTURE.md` は現在76KB超（352行）に肥大化しており、AI-IDEのコンテキストウィンドウを大きく圧迫している。本資料の真の目的は「AIがどの責務を修正する際にどのファイルを開くべきかを示す道標（全体マップ／索引）」であるが、現状はメソッドの内部ロジック・引数・ConfigManagerの全パラメータ羅列など、コードを直接読めば瞬時に理解できる詳細が大量に記載されている。

本フェーズでは、これらの「ノイズ」をごっそり削除し、資料を極限までスリム化する。
