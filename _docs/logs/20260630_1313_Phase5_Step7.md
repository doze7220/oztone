# 実装レポート：Phase 5 - Step 7 (先読みデータを活用した「次の曲」表示)

## 実装目的
先読み（プリフェッチ）されたデータを活用し、ウィンドウ右下を基準点としたレイアウトで「次の曲（極小アルバムアート＋曲名）」をUI上に描画する。ロード中（先読み中）の場合は代替の四角板を表示する機能を実装。

## 変更ファイル
*   `src/ConfigManager.h` / `src/ConfigManager.cpp`
    *   `[Layout_NextTrack]` セクションに対する設定値の読み書き処理を追加。
    *   指定の通り、`BaseRightOffset`, `BaseBottomOffset`, `ArtOffsetX`, `ArtOffsetY`, `ArtSize`, `BgOpacity`, `FallbackArtOpacity`, `TitleOffsetX`, `TitleOffsetY`, `TitleFontSize`, `ArtistOffsetX`, `ArtistOffsetY`, `ArtistFontSize` をサポート。
*   `src/Renderer.h` / `src/Renderer.cpp`
    *   `SetNextTrackInfo` メソッドを追加し、「次の曲」の画像と文字列を保持するメンバ変数を追加。
    *   `Render()` にて、ウィンドウ論理サイズから `BaseRightOffset`, `BaseBottomOffset` を使って右下基準の座標を算出し、ロード状態に応じた描画（代替プレースホルダー、または正規画像とテキスト）を追加。
    *   「次の曲」用のフォントサイズに合わせた `IDWriteTextFormat` を新規作成（タイトル用 14.0f、アーティスト用 12.0f）。
*   `src/Application.cpp`
    *   `Run()` ループ内で描画直前に、毎フレーム `m_renderer.SetNextTrackInfo()` に `m_isPrefetchReady`, `m_prefetchedAlbumArt`, `m_prefetchedTitle`, `m_prefetchedArtist` を渡して状態を同期。
*   `PROJECT_ARCHITECTURE.md`
    *   UI仕様方針に「右下：次の曲の極小アルバムアートと曲名…」を追記。

## 懸念点・備考
*   `ConfigManager` の `Layout_NextTrack` セクション内のオフセットやサイズについて、とりあえずリクエストのあった初期パラメータを反映していますが、起動後のレイアウト調整が必要になる可能性があります（特にウィンドウサイズを変えた場合や右下の配置バランスなど）。ユーザーにて `OZtone.ini` を直接編集して調整することを前提としています。

## 追加作業 (追記)
*   曲名に付与していた「Next: 」プレフィックスを削除し、完全に独立した「Next Track」ラベルとして描画するよう変更しました。
*   このラベルの位置（`LabelOffsetX`, `LabelOffsetY`）とフォントサイズ（`LabelFontSize`）を `ConfigManager` および `OZtone.ini` から設定できるように拡張しました。
